### 9. ORB特征提取的缺陷及如何进行改进

- 主标签：多模态与视觉语言
- 来源条数：1
- 答案生成方式：近似题匹配：DeepLearing-Interview-Awesome-2024_视觉感知专题:VisionPerception/Reference.md
- 来源：[DeepLearing-Interview-Awesome-2024 / 原仓库题解 / 未知](https://github.com/315386775/DeepLearing-Interview-Awesome-2024/blob/main/VisionPerception/Reference.md)

### 基础知识补充

- ORB对尺度变化敏感，缺乏真正的尺度不变性。
- 旋转不变性依赖灰度质心法，大角度下精度下降。
- 提取的特征点容易扎堆，导致局部信息冗余。

### 详细解答

结论：ORB特征提取的主要缺陷在于尺度不变性较弱、特征点分布不均以及对剧烈光照和视角变化的鲁棒性不足。 原理：ORB结合了FAST角点和BRIEF描述子。虽然通过构建图像金字塔在一定程度上缓解了尺度问题，但相比SIFT，其尺度不变性依然较差；其旋转不变性通过灰度质心法实现，但在复杂纹理或大视角倾斜时计算不准。此外，FAST角点容易在纹理丰富区域密集出现，导致特征点扎堆。 改进方案：针对尺度问题，可增加金字塔层数或结合尺度空间极值检测；针对分布不均，常采用四叉树（QuadTree）划分或网格化非极大值抑制（NMS）来均匀化特征点；针对光照和视角变化，可引入仿射变换模拟或直接使用基于深度学习的特征提取算法（如SuperPoint）来替代传统ORB。

### 案例模拟

面试官追问：在SLAM系统中，如何解决ORB特征点扎堆的问题？ 回答：在ORB-SLAM2/3中，作者采用了四叉树（Quad-Tree）结构来均匀化特征点分布。具体做法是将图像不断划分为四个子节点，如果某个节点内的特征点数量超过阈值，则继续划分；最后在每个网格内保留响应值（Harris角点响应）最大的特征点。这种工程处理不仅保证了特征点在全图的均匀覆盖，还能有效控制特征点总数，提升实时性。

### 22. 二、blip2的架构，优势和之前多模态模型的区别？

- 主标签：多模态与视觉语言
- 来源条数：1
- 答案生成方式：模型自动补全

### 基础知识补充

- 引入轻量级Q-Former连接冻结的视觉和语言模型
- 分两阶段预训练：视觉语言表示学习与生成学习
- 极大降低了多模态大模型的训练计算成本

### 详细解答

结论：BLIP-2 是一种高效的多模态预训练架构，其核心创新在于引入了 Q-Former 模块，成功桥接了冻结的预训练图像编码器和冻结的大语言模型（LLM），实现了极低成本的多模态对齐。 原理：与之前需要端到端重训整个模型的方法不同，BLIP-2 保持视觉和语言大模型参数冻结。Q-Former 包含一组可学习的 Query 向量，通过交叉注意力机制从视觉编码器中提取与文本相关的视觉特征。训练分为两阶段：第一阶段 Q-Former 与冻结的视觉模型进行对比学习、图文匹配和图像描述生成；第二阶段将 Q-Former 的输出作为前缀 Prompt 输入到冻结的 LLM 中，训练文本生成能力。 对比与权衡：相比于 Flamingo 或 CLIP，BLIP-2 的优势在于“站在巨人的肩膀上”，复用了现有 LLM 的强大推理能力，训练参数量极小。但由于视觉特征被压缩为固定数量的 Query，可能在处理极度细粒度的视觉细节时存在信息丢失。

### 案例模拟

面试官追问：BLIP-2中的Q-Former为什么要设计成两阶段训练？ 回答：两阶段训练是为了逐步缩小模态间的鸿沟。第一阶段强制Q-Former学习提取与文本强相关的视觉特征，过滤掉无关的背景信息，这相当于做特征对齐；第二阶段则是将这些对齐后的视觉特征转换为LLM能够理解的“视觉提示（Visual Prompt）”。如果不经过第一阶段直接连入LLM，LLM很难从随机初始化的Query中解析出有意义的视觉信息，导致收敛困难。

### 32. 如何理解concat和add这两种常见的feature map特征融合方式?

- 答案生成方式：原仓库答案
- 来源：[DeepLearing-Interview-Awesome-2024 / 原仓库题解 / 未知](https://github.com/315386775/DeepLearing-Interview-Awesome-2024/blob/main/VisionPerception/Reference.md)

- 先讲模态对齐
- 再讲编码器设计
- 补训练与推理成本

### 详细解答

两者都可以理解为整合特征图信息。concat是通道数的增加;add是特征图相加，通道数不变。 add是描述图像的特征下的信息量增多了，但是描述图像的维度本身并没有增加，只是每一维下的信息量在增加，这显然是对最终的图像的分类是有益的。而concat是通道数的合并，也就是说描述图像本身的特征数（通道数）增加了，而每一特征下的信息是没有增加。concat每个通道对应着对应的卷积核。 而add形式则将对应的特征图相加，再进行下一步卷积操作，相当于加了一个先验：对应通道的特征图语义类似，从而对应的特征图共享一个卷积核（对于两路输入来说，如果是通道数相同且后面带卷积的话，add等价于concat之后对应通道共享同一个卷积核）。因此add可以认为是特殊的concat形式。但是add的计算量要比concat的计算量小得多。
