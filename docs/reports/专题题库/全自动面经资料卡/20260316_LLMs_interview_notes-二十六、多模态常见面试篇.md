## 多模态常见面试篇

### 1. 一、最近关注的论文，多模态视觉大模型(CLIP,DALLE)？

- 主标签：多模态与视觉语言
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 多模态常见面试篇 / 未知](https://articles.zsxq.com/id_hmoqafrxjumk.html)

### 基础知识补充

- CLIP通过图文对比学习实现零样本图像分类
- DALL-E基于离散VAE和自回归Transformer生成图像
- 多模态对齐是当前视觉语言大模型的核心研究方向

### 详细解答

结论：近期多模态视觉大模型领域发展迅速，CLIP和DALL-E是该领域的奠基性工作。CLIP解决了视觉与语言特征空间的对齐问题，而DALL-E则开创了基于文本提示生成高质量图像的先河。 原理：CLIP（Contrastive Language-Image Pretraining）使用海量图文对，通过对比学习拉近匹配图文的特征距离，推远不匹配的距离，赋予了模型强大的Zero-shot迁移能力。DALL-E则先用dVAE将图像压缩为离散的视觉Token，然后将文本Token与视觉Token拼接，利用自回归Transformer进行联合建模，实现文生图。 工程权衡：CLIP的对比学习需要极大的Batch Size（如32768）以提供足够的负样本，对显存和分布式通信要求极高；而DALL-E的自回归生成速度较慢，这也是后来扩散模型（如Stable Diffusion）逐渐取代自回归生成模型的原因之一。

### 案例模拟

面试官追问：CLIP的图文特征对齐在实际业务中有什么应用场景？ 回答：CLIP在业务中有非常广泛的应用。例如在电商场景中，我们可以利用CLIP提取商品图片的视觉特征和用户搜索词的文本特征，通过计算余弦相似度实现“以文搜图”或“以图搜图”的跨模态检索。此外，CLIP的特征也常作为下游任务（如图像打标、视频内容理解）的强大预训练特征提取器。

### 2. 二、blip2的架构，优势和之前多模态模型的区别？

- 主标签：多模态与视觉语言
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 多模态常见面试篇 / 未知](https://articles.zsxq.com/id_hmoqafrxjumk.html)

### 基础知识补充

- 引入轻量级Q-Former连接冻结的视觉和语言模型
- 分两阶段预训练：视觉语言表示学习与生成学习
- 极大降低了多模态大模型的训练计算成本

### 详细解答

结论：BLIP-2 是一种高效的多模态预训练架构，其核心创新在于引入了 Q-Former 模块，成功桥接了冻结的预训练图像编码器和冻结的大语言模型（LLM），实现了极低成本的多模态对齐。 原理：与之前需要端到端重训整个模型的方法不同，BLIP-2 保持视觉和语言大模型参数冻结。Q-Former 包含一组可学习的 Query 向量，通过交叉注意力机制从视觉编码器中提取与文本相关的视觉特征。训练分为两阶段：第一阶段 Q-Former 与冻结的视觉模型进行对比学习、图文匹配和图像描述生成；第二阶段将 Q-Former 的输出作为前缀 Prompt 输入到冻结的 LLM 中，训练文本生成能力。 对比与权衡：相比于 Flamingo 或 CLIP，BLIP-2 的优势在于“站在巨人的肩膀上”，复用了现有 LLM 的强大推理能力，训练参数量极小。但由于视觉特征被压缩为固定数量的 Query，可能在处理极度细粒度的视觉细节时存在信息丢失。

### 案例模拟

面试官追问：BLIP-2中的Q-Former为什么要设计成两阶段训练？ 回答：两阶段训练是为了逐步缩小模态间的鸿沟。第一阶段强制Q-Former学习提取与文本强相关的视觉特征，过滤掉无关的背景信息，这相当于做特征对齐；第二阶段则是将这些对齐后的视觉特征转换为LLM能够理解的“视觉提示（Visual Prompt）”。如果不经过第一阶段直接连入LLM，LLM很难从随机初始化的Query中解析出有意义的视觉信息，导致收敛困难。
