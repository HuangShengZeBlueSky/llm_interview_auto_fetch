### 1. 一、谈谈你对当前出现的各种大模型的见解？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 闭源模型如GPT-4在通用能力和复杂推理上保持领先
- 开源模型如Llama系列推动了生态繁荣和垂直领域落地
- 行业趋势正向多模态长上下文和端侧轻量化方向发展

### 详细解答

当前大模型生态呈现出“闭源引领上限，开源繁荣生态”的格局。结论上，大模型已经从单纯的参数竞赛转向了综合能力的工程化落地。闭源模型（如GPT-4、Claude 3）凭借庞大的算力和数据优势，在复杂逻辑推理、代码生成和多模态理解上依然占据统治地位，适合作为通用Agent的大脑。而开源模型（如Llama 3、Qwen、Mistral）的性能正在快速逼近GPT-3.5甚至GPT-4，极大地降低了企业的研发门槛，使得在垂直领域（如医疗、金融）进行私有化部署和微调成为可能。未来的发展趋势将集中在三个方面：一是原生多模态能力的融合；二是支持超长上下文窗口（如百万Token）；三是MoE架构和量化技术推动的端侧小模型普及。

### 案例模拟

面试官追问：在企业实际落地中，你会如何选择开源还是闭源模型？ 回答：这取决于业务场景的数据敏感度和任务复杂度。如果是内部知识库问答或涉及敏感用户数据的场景，我会优先选择开源模型（如Qwen-14B）进行本地部署和微调，以保证数据安全。如果是需要极高推理能力、且数据不敏感的复杂规划任务，我会直接调用GPT-4或Claude的API，以降低自研成本并保证效果。

### 2. 二、目前大模型常见的 base 模型训练和 chat 模型训练 方式 的区别么？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- Base模型通过海量无标签数据进行自监督预训练
- Chat模型在Base基础上进行指令微调和人类对齐
- Base模型目标是续写文本而Chat模型目标是遵循指令

### 详细解答

Base模型和Chat模型的训练方式在目标、数据和范式上存在本质区别。结论上，Base模型是打地基，赋予模型世界知识；Chat模型是精装修，赋予模型交互能力。Base模型（预训练阶段）使用海量、多样化的无标签语料（如网页、书籍），采用自监督的下一个词预测（Next Token Prediction）任务，目的是让模型学习语言规律和通用知识，其输出表现为无意识的文本续写。Chat模型（对齐阶段）则在Base模型基础上，先通过SFT（监督微调）使用高质量的人工标注问答对，让模型学会理解指令和对话格式；随后通过RLHF（基于人类反馈的强化学习）或DPO等技术，使模型的输出符合人类的价值观和偏好（如诚实、无害、有用）。

### 案例模拟

面试官追问：如果直接用Base模型做问答，会出现什么问题？ 回答：直接使用Base模型做问答，它通常不会直接回答问题，而是倾向于续写问题。例如，输入“中国的首都是哪里？”，Base模型可能会续写“美国的首都是哪里？日本的首都是哪里？”，因为它只是在模仿训练数据中常见的问卷格式，缺乏“回答指令”的意识。必须通过Few-shot提示才能勉强引导其回答。

### 3. 3.1.1.1 llama 训练数据 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 训练数据完全来自公开可用的开源数据集
- 包含CommonCrawl维基百科GitHub等多个来源
- 数据经过严格的去重清洗和语言过滤处理

### 详细解答

LLaMA 1的训练数据最大的亮点在于其完全依赖公开可用的开源数据集，证明了不使用专有数据也能训练出顶尖模型。结论上，这种开源精神极大地推动了学术界的研究。其训练语料总计约1.4T Tokens，主要来源包括：经过大规模过滤和去重的CommonCrawl（占比67%）、C4数据集（15%）、GitHub代码（4.5%）、Wikipedia多语言词条（4.5%）、Gutenberg和Books3的开源书籍（4.5%）、ArXiv学术论文（2.5%）以及StackExchange问答（2%）。在工程处理上，Meta团队对数据进行了严格的清洗，包括使用FastText过滤非英语网页、使用n-gram去重，并训练了一个线性分类器来剔除低质量页面，确保了模型输入的高信噪比。

### 案例模拟

面试官追问：LLaMA在训练数据配比上有什么值得借鉴的地方？ 回答：LLaMA在数据配比上非常注重多样性和高质量知识的注入。虽然CommonCrawl占了大头，但他们刻意提高了Wikipedia、书籍和学术论文的采样权重（Epochs大于1），这意味着模型在训练时会多次阅读这些高质量的事实性文本。这种策略有效提升了模型在常识推理和知识问答任务上的表现。

### 4. 3.1.1.2 llama 模型参数量 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- LLaMA1包含7B、13B、33B和65B四个参数版本
- 7B和13B版本旨在实现单张消费级显卡上的推理
- 遵循Chinchilla缩放定律使用更多Token训练小模型

### 详细解答

LLaMA 1系列模型提供了7B、13B、33B和65B四个不同参数规模的版本，旨在满足不同计算资源下的研究和应用需求。结论上，LLaMA的设计哲学是“用更多的训练数据训练相对较小的模型”，以优化推理成本。根据DeepMind提出的Chinchilla缩放定律，10B左右的模型通常只需要约200B的Token即可达到训练最优，但Meta在训练LLaMA 7B和13B时，使用了高达1万亿（1T）的Tokens，而33B和65B版本则使用了1.4万亿Tokens。这种“过度训练（Over-training）”的工程权衡使得LLaMA 13B在大多数基准测试中超越了参数量大十倍的GPT-3（175B），并且7B/13B模型经过量化后，可以轻松部署在单张消费级GPU（如RTX 3090）甚至手机端。

### 案例模拟

面试官追问：为什么现在开源界特别钟爱7B到14B这个参数量级？ 回答：主要是出于部署成本的考量。7B模型在FP16精度下占用约14GB显存，INT8量化后仅需7GB左右，完美契合了市面上主流的24G显存显卡（如4090/3090）。这个参数量级既能保证模型具备一定的涌现能力和逻辑推理能力，又使得个人开发者和中小企业能够以极低的硬件门槛进行微调和本地部署。

### 5. 3.1.1.3 llama 模型结构 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 采用标准的Transformer解码器架构进行优化
- 使用RMSNorm进行前置归一化提升训练稳定性
- 引入RoPE旋转位置编码和SwiGLU激活函数

### 详细解答

LLaMA在模型结构上沿用了经典的Transformer Decoder-only架构，但吸收了近年来多项前沿优化技术以提升性能和训练稳定性。结论上，LLaMA的结构设计成为了后来众多开源大模型的“事实标准”。具体改进包括：1) 归一化：放弃了传统的LayerNorm，改用RMSNorm，并采用Pre-normalization（前置归一化）策略，去除了均值计算，提高了计算效率和训练稳定性；2) 激活函数：使用SwiGLU替换了原有的ReLU，虽然增加了参数量，但显著提升了模型的表达能力；3) 位置编码：移除了绝对位置编码，采用了RoPE（旋转位置编码），这种编码方式巧妙地结合了绝对位置和相对位置的优势，使得模型在处理长文本时具有更好的外推性。

### 案例模拟

面试官追问：RoPE（旋转位置编码）相比绝对位置编码有什么优势？ 回答：绝对位置编码在训练时固定了最大长度，难以泛化到更长的文本。而RoPE通过在复数空间中对词向量进行旋转，使得两个Token之间的内积仅依赖于它们的相对距离。这种特性不仅保留了绝对位置信息，还天然具备处理相对位置的能力，使得模型在推理时能够更好地外推到超过训练长度的上下文中。

### 6. 3.1.1.4 llama 训练目标 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 采用标准的自回归下一个词预测作为训练目标
- 优化目标是最小化交叉熵损失函数
- 训练过程中不包含任何掩码语言模型任务

### 详细解答

LLaMA的训练目标非常纯粹，采用了标准的自回归（Auto-regressive）语言建模任务，即“下一个词预测（Next Token Prediction）”。结论上，这是目前生成式大模型最通用且被证明最有效的预训练范式。在原理上，给定一个由词元组成的文本序列，模型的目标是基于前面的所有词元，最大化当前词元出现的条件概率。在工程实现中，这等价于最小化模型预测的概率分布与真实标签之间的交叉熵损失（Cross-Entropy Loss）。与BERT等采用掩码语言模型（MLM）双向理解的架构不同，LLaMA作为Decoder-only模型，通过因果掩码（Causal Mask）确保在预测第i个词时，只能看到1到i-1的词，从而赋予了模型强大的文本生成能力。

### 案例模拟

面试官追问：为什么现在的大模型几乎都放弃了BERT那种MLM训练目标？ 回答：MLM（掩码语言模型）虽然在双向上下文理解（如文本分类、实体识别）上表现优异，但它在训练和推理时存在差异（推理时没有[MASK]标记），且不擅长开放式的文本生成。而自回归目标天然契合人类说话和写作的顺序，随着模型规模的扩大，自回归模型不仅在生成任务上表现卓越，其通过Few-shot也能很好地完成理解类任务，实现了能力的统一。

### 7. 3.1.1.5 llama tokenizer 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 采用BPE字节对编码算法进行分词处理
- 使用SentencePiece工具库实现词表大小为32000
- 将所有数字拆分为单独的数字位以增强数学能力

### 详细解答

LLaMA在分词器（Tokenizer）的设计上采用了基于字节对编码（BPE, Byte-Pair Encoding）的算法，并使用SentencePiece库进行实现。结论上，这种设计兼顾了词汇覆盖率和序列长度的平衡。LLaMA 1的词表大小被设定为32,000，这是一个相对紧凑的词表，有助于控制嵌入层的参数量。在工程细节上，LLaMA的分词器有两个显著特点：一是它会将所有未知的UTF-8字符回退到字节级别进行编码，这意味着它理论上可以处理任何罕见字符而不会出现[UNK]（未知标记）；二是它将所有的数字拆分为单独的数字位（例如将"123"拆分为"1"、"2"、"3"），这种策略极大地缓解了模型在算术运算时因数字组合过多而导致的混乱，提升了数学推理能力。

### 案例模拟

面试官追问：LLaMA原生的Tokenizer对中文支持好吗？如何解决？ 回答：LLaMA原生的32K词表中，中文字符非常少（仅几百个），导致中文文本会被切分成大量的字节片段，不仅编码效率极低（一个汉字可能占3个Token），还影响了中文语义的表达。在实际项目中，我们通常会进行“词表扩充”，即在原有词表基础上，利用中文语料训练一个新的BPE词表并合并，然后冻结原模型参数，仅微调Embedding层，从而大幅提升中文处理能力。

### 8. 3.1.1.6 llama 衍生模型 介绍

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- Alpaca通过Self-Instruct技术微调LLaMA实现对话能力
- Vicuna利用ShareGPT高质量对话数据进行微调
- 中文社区推出了Chinese-LLaMA等扩充词表的版本

### 详细解答

LLaMA 1的开源直接引爆了开源大模型生态，催生了大量优秀的衍生模型。结论上，这些衍生模型主要通过指令微调（SFT）和词表扩充，弥补了原版LLaMA缺乏对话能力和多语言支持的短板。斯坦福大学推出的Alpaca是早期的代表作，它利用OpenAI的API通过Self-Instruct技术生成了5.2万条指令数据对LLaMA 7B进行微调，证明了极低成本也能激发大模型的对话能力。随后，LMSYS团队推出的Vicuna，使用了从ShareGPT收集的7万条高质量真实用户对话数据进行微调，在多轮对话和逻辑推理上表现更为出色。在国内，为了解决中文支持问题，研究者推出了Chinese-LLaMA和Ziya等模型，通过扩充中文词表并在中文语料上进行增量预训练，极大地推动了中文大模型的落地。

### 案例模拟

面试官追问：Alpaca使用的Self-Instruct技术具体是怎么做的？ 回答：Self-Instruct是一种利用强模型（如GPT-3.5）自动生成训练数据的方法。具体来说，先人工编写少量（如175个）种子指令，然后将这些种子输入给强模型，提示它生成更多类似的新指令和对应的解答。通过这种方式，可以低成本地构造出数万条多样化的指令微调数据集，用于训练较小的开源模型，这在早期极大地缓解了高质量SFT数据匮乏的问题。

### 9. 3.2.1 llama2 系列 数据预处理方式？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 预训练语料规模提升至2万亿Tokens
- 增加了事实性数据源的权重以减少幻觉
- 剔除了包含大量个人隐私信息的站点数据

### 详细解答

LLaMA 2在数据预处理上相比第一代进行了更为严格和精细的工程优化，核心目标是提升模型的安全性和事实准确性。结论上，高质量的数据清洗是LLaMA 2性能飞跃的关键。在预训练阶段，LLaMA 2的语料规模从1.4T增加到了2T Tokens。在数据处理方式上，Meta团队不仅进行了常规的去重和语言过滤，还特别针对安全性进行了优化：主动剔除了已知包含大量个人身份信息（PII）的网站数据，以降低隐私泄露风险。此外，为了提升模型的事实性并减少幻觉，团队在上采样阶段增加了高度事实性数据源（如维基百科、高质量学术论文）的权重。在微调阶段，LLaMA 2-Chat更是引入了超过100万条人类偏好数据进行RLHF训练。

### 案例模拟

面试官追问：为什么在预训练阶段要剔除包含个人隐私的数据？ 回答：大模型具有很强的“记忆效应”，如果在预训练语料中包含了用户的真实姓名、电话、住址等隐私信息，模型在生成时极有可能将其原样输出，造成严重的法律和合规风险。剔除这些数据是从源头上保证模型安全性的必要手段，比在推理阶段使用规则拦截更加彻底和有效。

### 10. 3.2.2 llama2 系列 Tokenizer 处理方式？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / LLMs 对比篇 / 未知](https://articles.zsxq.com/id_fsq8czgwjxse.html)

### 基础知识补充

- 沿用LLaMA1的BPE算法和32000词表大小
- 同样将数字拆分为单一位并回退到字节级别
- 增加了特殊的控制字符用于Chat版本的对话格式

### 详细解答

LLaMA 2在Tokenizer（分词器）的核心设计上完全沿用了LLaMA 1的方案，这保证了生态的延续性。结论上，它依然采用基于SentencePiece的BPE算法，词表大小保持为32,000。在处理逻辑上，同样保留了将所有数字拆分为独立数字位（提升数学能力）以及将未知字符回退到Byte级别（避免UNK标记）的优秀特性。然而，在LLaMA 2-Chat版本中，为了支持更复杂的对话结构和系统指令，Tokenizer在应用层面引入了一套特殊的控制字符（如[INST]、[/INST]、>等）。这些特殊Token在数据预处理阶段被插入到文本中，用于明确区分系统提示词、用户输入和模型回答，从而帮助模型更好地理解多轮对话的上下文边界。

### 案例模拟

面试官追问：如果我想在LLaMA 2上微调一个多轮对话模型，数据格式应该怎么处理？ 回答：必须严格遵循LLaMA 2-Chat的特殊Token格式。具体来说，系统提示词需要用>和>包裹，用户的每一轮输入需要用[INST]和[/INST]包裹。如果格式拼接错误，模型在推理时会因为无法识别对话边界而产生混乱的输出或无法正常停止生成。

### 11. 3.2.3 llama2 系列 Architectural？

面试官追问：空白填充任务与传统的 Next Token Prediction 有何异同？回答：Next Token Prediction 只能从左到右单向预测，而空白填充允许模型利用被掩码片段前后的双向上下文信息进行预测。这使得 GLM 架构在处理填空、摘要等任务时具有天然优势，同时通过自回归生成掩码内容，保留了强大的生成能力。

### 详细解答

BLOOM 模型的结构基于经典的 Decoder-only Transformer 架构，但在细节上进行了针对超大模型的优化。最显著的改进是采用了 ALiBi（Attention with Linear Biases）位置编码。与传统的绝对或相对位置 Embedding 不同，ALiBi 直接在注意力分数矩阵上加上一个与距离成正比的负偏置，这使得模型不仅训练速度更快，而且具备极强的长度外推能力，能够在推理时处理比训练时更长的文本。此外，为了解决 176B 超大模型训练中的不稳定性问题，BLOOM 在词嵌入层（Embedding Layer）之后额外增加了一个 LayerNorm 层。这种设计有效防止了早期训练阶段梯度爆炸的问题，确保了千亿参数模型的平稳收敛。

### 案例模拟

面试官追问：为什么在 Embedding 后加 LayerNorm 能稳定训练？回答：在千亿规模模型中，Embedding 层的输出范数容易在训练初期急剧变大，导致后续层的激活值和梯度失控。在 Embedding 后添加 LayerNorm 可以强制将输入特征归一化到合理的分布范围内，从而为深层网络提供稳定的输入信号，避免数值溢出。

纵观各大主流大语言模型的训练过程，存在着显著的共同点。首先，在数据层面，所有模型都极度依赖海量、高质量的无监督文本语料，且数据清洗（去重、过滤、脱敏）的精细度直接决定了模型的上限。其次，在训练目标上，绝大多数模型都采用了因果语言建模（Causal Language Modeling），即通过自回归方式预测下一个 Token。在工程实现层面，由于模型参数量动辄百亿甚至千亿，单张显卡无法容纳，因此普遍采用了 3D 并行技术（数据并行、张量并行、流水线并行）以及 ZeRO 优化器状态切分技术，以实现大规模 GPU 集群上的高效协同训练。此外，SwiGLU、RMSNorm 和 RoPE 等组件已成为标配。

### 30. 4.2 大模型训练不同点？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全

### 基础知识补充

- 位置编码选择不同（RoPE、ALiBi、2D RoPE）。
- 词表大小与多语言支持侧重点存在差异。
- 注意力机制优化不同（MHA、MQA、GQA）。

### 详细解答

尽管大模型在宏观思路上相似，但在具体技术选型上存在诸多不同点。首先是位置编码，LLaMA 和 Qwen 倾向于使用 RoPE，BLOOM 采用 ALiBi，而 ChatGLM 则针对其 Prefix Decoder 设计了 2D RoPE。其次，在 Tokenizer 设计上，不同模型的词表大小差异巨大（如 LLaMA 的 3.2 万对 BLOOM 的 25 万），这反映了它们在多语言支持和计算效率之间的不同权衡。再者，为了优化推理速度，注意力机制的实现也有所不同：早期模型多用标准多头注意力（MHA），而较新的模型（如 Mistral、LLaMA-2）则广泛采用多查询注意力（MQA）或分组查询注意力（GQA）以降低 KV Cache 的显存占用。

### 案例模拟

面试官追问：MQA 和 GQA 在工程实践中如何选择？回答：MQA 让所有 Query 共享一对 KV 头，推理速度最快但可能带来轻微的性能下降；GQA 则是将 Query 分组，每组共享一对 KV 头。GQA 是一种折中方案，在保持接近 MHA 模型性能的同时，实现了接近 MQA 的推理加速，是目前大多数新模型的首选。

- 后续的实际文本单向或双向依赖于这些前缀。

### 详细解答

结论是：在 Prefix-Tuning 中，Prefix Tokens 内部通常是双向注意力（互相可见），而后续文本对它们的注意力方式取决于基础模型的架构。Prefix-Tuning 的核心思想是在 Transformer 的每一层前面拼接一组可学习的连续向量（Prefix Tokens）。如果基础模型是像 BERT 这样的 Encoder 架构，整个序列（包括 Prefix）都是双向注意力；如果基础模型是像 GPT 这样的 Decoder 架构，Prefix Tokens 自身作为上下文的前缀，它们之间可以互相看到（即不受因果掩码限制），而后续的实际输入文本则通过单向注意力（因果掩码）依赖于这些 Prefix Tokens。这种设计使得 Prefix 能够充分编码全局的任务指令信息，从而高效引导模型生成。

### 案例模拟

面试官追问：Prefix-Tuning 和 Prompt Tuning 有什么区别？回答：Prompt Tuning 仅在模型的输入层（Embedding 层）添加可学习的虚拟 Token；而 Prefix-Tuning 是在 Transformer 的每一层（包括多头注意力的 Key 和 Value）都添加可学习的前缀向量。因此，Prefix-Tuning 的参数量稍大，但表达能力更强，通常在复杂任务上效果更好。

### 39. 一、gpt源码past\_key\_value是干啥的？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- 用于缓存历史Token的Key和Value张量。
- 避免自回归生成时重复计算前面的上下文。
- 是大模型推理加速的核心机制（KV Cache）。

### 详细解答

在GPT等自回归语言模型的源码中，past_key_value（通常被称为KV Cache）是用于大幅提升推理效率的关键机制。在文本生成过程中，模型需要逐个预测下一个Token。如果没有缓存机制，每次预测新Token时，模型都需要将之前生成的所有Token重新输入Transformer层进行完整的Attention计算，这会导致大量的冗余计算，且计算复杂度随序列长度呈平方级增长。 past_key_value的作用就是在每一步生成时，将当前Token在各层Attention中计算得到的Key（K）和Value（V）向量保存下来。在预测下一个Token时，模型只需计算当前新Token的Q、K、V，然后将新的K、V与缓存的past_key_value拼接，即可完成Attention计算。这使得每步生成的计算复杂度变为线性，极大地降低了延迟。

### 案例模拟

面试官追问：KV Cache会带来什么工程上的挑战？如何优化？ 回答：KV Cache最大的挑战是显存占用。随着Batch Size和上下文长度的增加，KV Cache会消耗巨大的显存，甚至超过模型权重本身，导致OOM。优化方案包括：算法层面采用MQA或GQA减少KV头数；工程层面采用PagedAttention（如vLLM）进行显存分页管理，减少显存碎片；或者对KV Cache进行量化（如INT8/INT4）存储。

### 40. 二、gpt onebyone 每一层怎么输入输出？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- 输入为Token的嵌入向量加上位置编码。
- 每层包含自注意力机制和前馈神经网络。
- 最后一层输出经过线性层和Softmax得到词表概率。

### 详细解答

GPT模型采用Decoder-only架构，其逐层（one-by-one）的输入输出流程如下：首先，输入的Token序列经过Embedding层转化为词向量，并加上位置编码（如绝对位置或RoPE），形成初始的隐藏状态（Hidden States）。 接着，这个隐藏状态依次穿过多个相同的Transformer Block。在每一层Block中，输入首先经过LayerNorm，然后进入掩码多头自注意力层（Masked Multi-Head Attention），计算得到上下文表示后，通过残差连接与原输入相加；随后再次经过LayerNorm，进入前馈神经网络（FFN），同样通过残差连接输出。这一层的输出直接作为下一层的输入。 当经过所有Transformer层后，最终的隐藏状态会通过一个全连接层（LM Head），将其映射到词表大小的维度，最后通过Softmax函数转化为预测下一个Token的概率分布。

### 案例模拟

面试官追问：在推理阶段，每一层的输入输出和训练阶段有什么不同？ 回答：训练阶段采用Teacher Forcing，整个序列一次性输入模型，利用并行计算和Causal Mask同时计算所有位置的Loss。而在推理阶段（自回归生成），模型是逐个Token生成的。每次只将最新生成的一个Token输入模型，在每一层中结合KV Cache（历史Token的K和V）进行计算，输出预测的下一个Token，循环往复。

### 41. 三、bert和gpt有什么区别

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- BERT是仅编码器架构，GPT是仅解码器架构。
- BERT采用双向注意力，GPT采用单向因果注意力。
- BERT适合自然语言理解，GPT适合自然语言生成。

### 详细解答

BERT和GPT是自然语言处理领域的两大里程碑模型，它们在架构设计和适用场景上存在根本区别。从架构上看，BERT（Bidirectional Encoder Representations from Transformers）采用的是Transformer的Encoder部分，而GPT（Generative Pre-trained Transformer）采用的是Decoder部分。 在注意力机制上，BERT使用双向自注意力，每个Token都能看到上下文的全部信息，这使得它非常适合文本分类、命名实体识别等自然语言理解（NLU）任务。而GPT使用带有掩码的单向因果注意力（Causal Attention），当前Token只能看到它之前的Token，这种自回归特性使其天生适合文本摘要、对话等自然语言生成（NLG）任务。此外，BERT的预训练任务是掩码语言模型（MLM）和下一句预测（NSP），而GPT则是标准的下一个词预测（Next Token Prediction）。

### 案例模拟

面试官追问：为什么现在的大语言模型（LLM）几乎都选择了GPT的Decoder-only架构，而不是BERT架构？ 回答：首先，Decoder-only架构的自回归生成任务（预测下一个词）难度更大，迫使模型学习到更深层的通用知识和推理能力；其次，Decoder-only在Zero-shot和Few-shot的Prompt学习上表现出极强的泛化能力，而BERT通常需要针对特定任务进行微调；最后，工程上Decoder-only的KV Cache机制使得生成效率更高。

- 序列到序列生成（Seq2Seq/Prefix LM）。

### 详细解答

在文本生成领域，预训练任务主要分为三大类，决定了模型的架构和适用场景。第一种是自回归语言建模（Auto-Regressive LM），代表模型是GPT系列。其任务是根据前面的上下文预测下一个词，采用严格的单向掩码，这种任务最符合人类生成文本的习惯，是目前大语言模型（LLM）的主流选择。 第二种是去噪自编码器任务（Denoising Autoencoder），代表模型是BART和T5。它通过对输入文本进行加噪（如掩码、删除、打乱顺序），然后要求模型在解码端重建原始文本。这种任务结合了双向理解和单向生成，非常适合翻译、摘要等Seq2Seq任务。 第三种是前缀语言建模（Prefix LM），代表模型是GLM。它允许模型对前缀部分进行双向注意力计算，而对生成部分进行单向自回归计算，试图兼顾理解和生成能力。

### 案例模拟

面试官追问：T5的Span Corruption任务具体是怎么做的？ 回答：T5的Span Corruption任务是掩码语言模型的一种变体。它会随机遮盖输入文本中连续的几个词（Span），并用一个特殊的哨兵Token（如）替换。在解码端，模型需要自回归地生成被遮盖的Span内容，每个Span前加上对应的哨兵Token。这种方式比单个词的掩码更能训练模型对长片段的重建能力。

### 43. 五、讲讲T5和Bart的区别，讲讲bart的DAE任务？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- T5和BART均采用标准的Encoder-Decoder架构。
- T5将所有NLP任务统一为文本到文本的生成。
- BART的DAE任务通过多种加噪方式重建原文。

### 详细解答

T5和BART都是经典的Encoder-Decoder架构模型，但在设计理念和预训练任务上有所不同。T5（Text-to-Text Transfer Transformer）的核心思想是“万物皆文本”，它将分类、翻译、摘要等所有NLP任务都转化为Text-to-Text的生成任务，并在输入前加上任务提示（如"translate English to German:"）。 BART则更侧重于去噪自编码器（DAE, Denoising Autoencoder）的预训练。BART的DAE任务通过对输入文本施加多种复杂的噪声，然后让Decoder重建原始干净文本。这些噪声包括：Token Masking（随机掩码）、Token Deletion（随机删除词，迫使模型学习位置信息）、Text Infilling（用单个掩码替换连续的文本片段）、Sentence Permutation（打乱句子顺序）和Document Rotation（随机选择一个词作为文章开头）。这种多样的加噪方式使BART在文本摘要和生成任务上表现极佳。

### 案例模拟

面试官追问：在做文本摘要任务时，你会选择BART还是T5？为什么？ 回答：在中小规模模型下，我倾向于选择BART。因为BART的预训练任务（如句子打乱、文本填充）天然契合文本摘要中需要的句子重组和信息提炼能力。而T5虽然通用性强，但其预训练任务（Span Corruption）更偏向于局部片段的填空。不过如果任务需要多任务混合处理，T5的Text-to-Text范式会更方便统一输入输出。

### 44. 六、讲讲Bart和Bert的区别？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- BERT是仅编码器架构，BART是编码器-解码器架构。
- BERT用于文本理解，BART用于文本生成和理解。
- BART的预训练加噪方式比BERT的MLM更丰富。

### 详细解答

BART和BERT虽然名字相似，但在架构和应用场景上有本质区别。BERT（Bidirectional Encoder Representations from Transformers）是一个纯Encoder架构，采用双向注意力机制。它的预训练任务是掩码语言模型（MLM），即预测被遮盖的词。BERT主要用于自然语言理解（NLU）任务，如文本分类、情感分析和序列标注，无法直接用于自回归文本生成。 BART（Bidirectional and Auto-Regressive Transformers）则是一个完整的Encoder-Decoder架构。它的Encoder像BERT一样是双向的，而Decoder像GPT一样是单向自回归的。BART的预训练任务是去噪自编码（DAE），不仅包含类似BERT的掩码，还包括删除、句子打乱等更复杂的噪声，要求Decoder重建原文。因此，BART不仅能做NLU任务，更擅长机器翻译、文本摘要等序列到序列（Seq2Seq）的生成任务。

### 案例模拟

面试官追问：BART的Encoder和BERT的Encoder在结构上完全一样吗？ 回答：基本结构非常相似，都是基于Transformer的Encoder层。但BART在激活函数上使用了GeLU，并且BART的Decoder在每一层都会通过Cross-Attention与Encoder的输出进行交互。此外，BART在预训练时没有像BERT那样使用Next Sentence Prediction (NSP) 任务，而是完全依赖于去噪重建任务。

### 45. 七、gpt3和gpt2的区别？

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / GPT 经验篇 / 未知](https://articles.zsxq.com/id_r46k6bqu34xh.html)

### 基础知识补充

- GPT-3参数量达1750亿，远超GPT-2的15亿。
- GPT-3引入了In-context Learning（上下文学习）。
- GPT-3在稀疏注意力机制上进行了优化。

### 详细解答

GPT-3和GPT-2在基础架构上都沿用了Transformer Decoder-only结构，但GPT-3在规模和能力上实现了质的飞跃。首先是规模的巨大差异，GPT-3的参数量达到了惊人的1750亿，是GPT-2（15亿）的100多倍，训练数据量也大幅增加。这种规模的扩张带来了显著的“涌现能力”。 其次，GPT-3最大的贡献是确立了In-context Learning（上下文学习）的范式。GPT-2虽然提出了Zero-shot的设想，但效果有限；而GPT-3证明了，只要模型足够大，无需进行梯度更新微调，仅通过在Prompt中提供几个示例（Few-shot），模型就能理解任务并输出高质量结果。此外，在网络结构细节上，GPT-3在部分Transformer层中引入了稀疏注意力机制（Sparse Attention），以降低超长上下文带来的计算复杂度。

### 案例模拟

面试官追问：GPT-3的In-context Learning和传统的Fine-tuning有什么本质区别？ 回答：本质区别在于是否更新模型权重。Fine-tuning需要使用特定任务的数据计算Loss并反向传播更新模型参数，这会改变模型的状态，甚至导致灾难性遗忘。而In-context Learning在推理阶段进行，模型权重保持冻结，它通过前向传播过程中的注意力机制，从Prompt中的示例动态提取任务模式，是一种“即用即走”的轻量级适配方式。
