### 2. ．讲一下大模型训练和推理的流程， SFT 和 RLHF 的作用分别是什么

- 主标签：模型微调
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- 训练流程：Pre-training -> SFT -> RLHF (RM+PPO/DPO)
- 推理流程：Prefill（预填充阶段） -> Decode（逐字生成阶段）
- SFT负责激发知识与格式对齐，RLHF负责价值观与偏好对齐

### 详细解答

结论：大模型的生命周期包含从海量无监督数据中学习世界知识的训练阶段，以及在实际应用中自回归生成文本的推理阶段。 原理：1）训练流程：首先是预训练（Pre-training），模型通过Next-token prediction在海量语料上学习语言规律和知识；接着是SFT（监督微调），使用高质量的问答对让模型学会遵循指令和特定格式；最后是RLHF，通过奖励模型和强化学习，让模型的输出符合人类的偏好（如无害、有用）。2）推理流程：分为Prefill阶段（并行计算输入Prompt的KV Cache）和Decode阶段（利用缓存的KV Cache，串行、逐个生成后续Token）。 工程权衡：SFT成本低但容易产生幻觉或迎合用户；RLHF能显著提升输出质量和安全性，但训练极不稳定，对超参数极其敏感。

### 案例模拟

面试官追问：在推理流程中，Prefill和Decode阶段的计算瓶颈分别是什么？如何针对性优化？ 回答：Prefill阶段的瓶颈是计算密集型（Compute-bound），因为需要一次性处理大量输入的矩阵乘法；而Decode阶段的瓶颈是访存密集型（Memory-bound），因为每次只生成一个Token，但需要频繁从显存中读取庞大的权重和KV Cache。针对性优化方面：对于Prefill，可以使用FlashAttention优化显存读写并提升计算效率；对于Decode，可以使用PagedAttention解决KV Cache的碎片化问题，或者引入投机解码（Speculative Decoding）通过小模型草稿和大模型验证来打破串行生成的瓶颈。

### 3. ．为什么探索 MoE 架构， MoE 相比 Dense 有什么好处

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- 稀疏激活机制：每次仅激活部分专家网络，降低计算开销。
- 路由网络（Router）：通过门控机制决定Token分配给哪些专家。
- 在保持推理算力不变的前提下，显著增加模型总参数量。

### 详细解答

结论：探索MoE（混合专家）架构的核心目的是打破模型参数量与计算算力之间的线性绑定关系。相比Dense（稠密）模型，MoE最大的好处是在相同的推理计算量下，能够拥有更大的参数规模，从而提升模型能力。 原理与对比：在Dense模型中，每个Token都需要经过所有参数的计算，导致参数量翻倍则计算量翻倍。而MoE引入了稀疏路由机制，将前馈神经网络（FFN）拆分为多个专家。推理时，Router会为每个Token动态选择Top-K（如Top-2）个专家进行激活。这样，虽然模型总参数量巨大（如百亿级别），但实际激活的参数量（即计算量）可能只有十分之一。 工程权衡：尽管MoE在计算效率上优势明显，但也带来了显存占用的挑战，因为所有专家的参数都需要常驻显存。此外，专家负载不均衡（某些专家被频繁调用，其他闲置）和通信开销（分布式训练时Token在不同GPU间路由）也是工程落地时需要重点优化的难题。

### 案例模拟

追问：MoE训练中常遇到的负载不均衡问题怎么解决？ 回答：通常会在损失函数中引入辅助损失（Auxiliary Loss）。它包含两部分：一是专家被分配到的Token比例，二是Router对各个专家的平均路由概率。通过最小化这两者的点积，强制Router将Token均匀分配给所有专家。在工程上，还可以采用Capacity Factor限制单个专家的最大处理量，超出部分直接丢弃或传递给下一层，从而保证训练效率。

### 4. ．有没有全量微调过

- 主标签：模型微调
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- 全参数微调（FFT）：更新模型所有权重以适配下游特定任务。
- 全量微调易导致模型丧失预训练阶段获得的通用能力。
- 常结合ZeRO分片、梯度检查点等技术降低训练显存。

### 详细解答

结论：全量微调（Full Fine-Tuning, FFT）是指在微调阶段放开大模型的所有参数进行梯度更新。相比于PEFT（如LoRA），全量微调能最大程度地让模型拟合目标领域的数据分布，但计算和显存成本极高。 原理与工程权衡：在实际业务中，全量微调通常用于模型需要进行深度领域适配、或者基础能力需要重塑的场景（如注入大量全新的垂直行业知识、改变模型的默认回复语言风格等）。由于需要保存所有参数的优化器状态和梯度，显存占用通常是模型参数量的数倍。因此，工程上必须依赖DeepSpeed的ZeRO-2/ZeRO-3技术进行显存状态分片，并配合Gradient Checkpointing（梯度检查点）以时间换空间。 对比：虽然FFT效果上限高，但极易引发“灾难性遗忘”，导致通用问答能力下降。因此，目前业界更多采用LoRA等高效微调方案，只有在数据量极大且算力充足的核心基座迭代时，才会谨慎使用全量微调。

### 案例模拟

面试官追问：如果全量微调7B模型，大概需要多少显存？怎么估算？ 回答：以AdamW优化器为例，7B模型参数本身占14GB（BF16）。优化器状态需保存动量和方差（FP32），占7B*8=56GB。梯度（FP32）占28GB。加上激活值，总显存约100GB以上。单张80G A100无法跑通，必须用多卡并开启ZeRO-2或ZeRO-3将优化器状态和梯度分片到不同GPU上，同时开启梯度检查点来大幅降低激活值显存占用。

### 5. ．在 RLHF 中，目前主流的强化学习算法有哪几个，写一下损失函数的表达式

- 主标签：后训练与对齐
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- PPO算法：近端策略优化，通过裁剪机制限制策略更新幅度。
- DPO算法：直接偏好优化，将RL目标转化为分类交叉熵损失。
- 奖励模型：评估生成回复质量，为强化学习提供标量奖励信号。

### 详细解答

结论：目前大模型RLHF主流的强化学习算法主要分为两派：以PPO（近端策略优化）为代表的传统Actor-Critic架构算法，以及以DPO（直接偏好优化）为代表的无奖励模型对齐算法。 原理与公式： 1. PPO的核心是限制策略更新步长，其Actor损失函数包含裁剪项：$L^{CLIP} = \min(r_t(\theta)\hat{A}_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_t)$，其中$r_t$是新旧策略的概率比，$\hat{A}_t$是优势函数。同时还会加入KL散度惩罚项防止偏离初始模型过远。 2. DPO则跳过了显式的奖励模型训练，直接在偏好数据上优化策略。其损失函数为：$L_{DPO} = -\log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right)$。 工程权衡：PPO上限高但训练极不稳定，需维护四个模型，显存开销巨大。DPO实现简单、显存占用少，是目前开源社区最常用的平替方案。

### 案例模拟

追问：PPO训练中KL散度惩罚项的作用是什么？如果系数设置过大或过小会怎样？ 回答：KL惩罚项用于约束当前策略模型（Actor）不要偏离参考模型（Ref）太远，防止模型为了迎合奖励模型而产生“奖励骇客（Reward Hacking）”现象，输出乱码或套话。如果KL系数过大，模型更新受限，难以学到人类偏好；如果过小，模型会过度优化奖励得分，导致生成文本失去自然语言的连贯性和多样性。

### 6. ．平常有用过 RLHF 吗，简单介绍一下

- 主标签：后训练与对齐
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- 监督微调：构建高质量指令集，让模型初步学会遵循用户指令。
- 偏好数据收集：人工或AI对模型生成的多个回复进行排序打分。
- 对齐税：对齐训练可能导致模型在部分基础通用任务上性能下降。

### 详细解答

结论：RLHF（基于人类反馈的强化学习）是大模型实现价值观对齐和指令遵循的核心技术范式。它通过引入人类偏好，使模型的输出更符合人类的期望（如诚实、无害、有用）。 原理与流程：标准RLHF通常包含三个阶段。第一阶段是SFT（监督微调），用高质量人工标注数据让模型具备基本的对话和指令遵循能力。第二阶段是训练奖励模型（RM），收集同一Prompt下的多个模型回复，由人类标注员进行排序，利用排序数据训练一个能给出标量分数的回归模型。第三阶段是强化学习（如PPO），利用RM给出的奖励信号，通过强化学习算法不断优化生成策略（Actor模型），同时用KL散度约束其不偏离SFT模型过远。 工程权衡：RLHF能显著提升主观体验，但数据标注成本极高，且PPO训练极不稳定。实际业务中，常使用AI反馈（RLAIF）代替人工，或采用DPO等更轻量化的算法来降低工程门槛。

### 案例模拟

项目模拟：在我们的垂直领域问答项目中，SFT后的模型虽然能回答专业问题，但语气生硬且容易产生幻觉。我们引入了轻量级的RLHF（采用DPO算法）。首先利用强模型（如GPT-4）对SFT模型的输出进行偏好打分，构建了1万条Chosen/Rejected数据对。经过DPO训练后，模型在保持专业性的同时，回复的亲和力和安全性显著提升，且避免了PPO复杂的四模型部署难题。

### 7. 开放题：对目前大模型的发展有什么看法

### 基础知识补充

- Scaling Law：算力与数据规模增加持续推动模型能力提升。
- 多模态融合：从纯文本向图文音视频的统一表征与原生生成演进。
- 端侧部署：借助量化与小模型架构，推动大模型在边缘设备落地。

### 详细解答

结论：目前大模型正处于从“基础能力突破”向“多模态融合与产业深水区落地”演进的关键阶段。整体发展呈现出“云端超大模型追求上限，端侧小模型追求性价比”的两极化趋势。 原理解释与趋势分析： 1. 架构与能力：Transformer依然是主流，但MoE架构已成为千亿/万亿模型的标配，以平衡算力与参数量。同时，长上下文（Long Context）能力和原生多模态（如Sora、GPT-4o）成为核心竞争点。 2. 训练范式：高质量数据的枯竭促使合成数据（Synthetic Data）和自我博弈（Self-Play）成为后训练阶段提升逻辑推理能力的关键。 3. 落地应用：Agent（智能体）技术是打通大模型与物理/软件世界的桥梁。 工程权衡：尽管模型能力不断增强，但算力成本、幻觉问题和数据隐私仍是商业化落地的三大阻碍。未来，通过RAG（检索增强）缓解幻觉，以及通过模型量化（如INT4）实现端侧部署，将是工程优化的主旋律。

面试官追问：你提到高质量数据即将枯竭，业界目前有什么解决思路？ 回答：目前主要有两个方向。一是利用强模型生成合成数据（Synthetic Data），例如用GPT-4生成复杂的逻辑推理链（CoT）或代码指令来训练开源小模型。二是探索基于自我对弈（Self-Play）的强化学习，让模型在特定环境（如数学证明、代码编译）中通过环境反馈的明确Reward进行自我迭代，减少对人工标注数据的依赖。

### 8. 代码：多头自注意力

- 主标签：LLM基础
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[真实面经_nowcoder / 腾讯大模型算法一面 / 腾讯](https://www.nowcoder.com/feed/main/detail/816b4b5d959f4df3ad6664e78b6593ee)

### 基础知识补充

- 线性映射：将输入特征分别投影为Q、K、V三个特征矩阵。
- 缩放点积：Q和K点积后除以维度平方根，防止进入梯度饱和区。
- 掩码机制：在自回归生成中，遮蔽未来位置以防止信息提前泄露。

### 详细解答

结论：多头自注意力（Multi-Head Attention, MHA）是Transformer架构的核心组件，它允许模型在不同的表示子空间中并行捕捉输入序列中不同位置的依赖关系。 原理与实现：MHA的核心思想是将高维的输入特征拆分成多个低维的“头（Head）”。具体步骤为：首先通过线性层将输入$X$映射为$Q, K, V$；然后将它们在隐藏维度上切分为$h$个头；接着对每个头独立计算缩放点积注意力：$\text{Attention}(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$。如果存在Mask（如Causal Mask），则在Softmax前将对应位置加上负无穷。最后，将所有头的输出在维度上拼接（Concat），并通过一个线性层进行特征融合。 工程权衡：标准MHA的计算复杂度和显存占用与序列长度的平方成正比。在长文本场景下，通常会采用FlashAttention技术进行硬件级IO优化，或替换为MQA/GQA架构以大幅减少KV Cache的显存占用。

### 案例模拟

追问：手写多头注意力时，Q和K转置相乘的维度变化是怎样的？ 回答：假设Batch Size为B，序列长度为L，头数为H，每个头的维度为D。切分多头后，Q和K的形状都是 (B, H, L, D)。在计算注意力分数时，需要对K的最后两个维度进行转置，变成 (B, H, D, L)。然后Q和转置后的K进行矩阵乘法（torch.matmul），得到形状为 (B, H, L, L) 的注意力分数矩阵，这代表了L个Token两两之间的相关性。
