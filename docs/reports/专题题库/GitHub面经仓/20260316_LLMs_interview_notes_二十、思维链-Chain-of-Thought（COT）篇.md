## 代表题答法

### 问题1：什么是思维链（CoT）及其核心思路？
**答题要点：**
1. **定义**：思维链（Chain-of-Thought）是一种提示技术，通过引导大语言模型在输出最终答案之前，显式地生成一系列中间推理步骤。
2. **核心思路**：模仿人类面对复杂问题时“分步思考”的过程。将原本复杂的直接映射（输入 $X \rightarrow$ 输出 $Y$），拆解为多个简单的、逻辑连贯的子步骤（输入 $X \rightarrow$ 推理 $Z_1 \rightarrow Z_2 \rightarrow$ 输出 $Y$）。
3. **应用场景**：在数学计算、常识推理、逻辑推导等需要多步计算的复杂任务中，能显著提升模型的准确率。

### 问题2：思维链提示的本质是什么？
**答题要点：**
1. **算力分配（Token即算力）**：大模型是自回归生成的，每个 Token 的生成都伴随着一次前向传播。CoT 通过强制模型输出中间步骤，实际上是为复杂问题分配了更多的“计算 Token”和思考时间，用空间（生成的长度）换取了准确度。
2. **概率分布优化**：在直接生成答案时，正确答案的条件概率可能很低。CoT 通过生成中间变量 $Z_i$，将复杂的联合概率分布拆解为多个高概率的条件概率连乘（即 $P(Y|X, Z)$ 通常远大于 $P(Y|X)$），从而降低了生成错误答案的概率。
3. **上下文激活**：中间推理步骤作为新的 Context 被输入给模型，能够更好地激活模型参数中相关的世界知识和逻辑规则，指导下一步的生成。

### 问题3：什么是 Zero-shot 和 Few-shot 提示方法？它们如何与 CoT 结合？
**答题要点：**
1. **Zero-shot（零样本提示）**：
   - **定义**：不给模型提供任何示例，直接输入任务描述和问题，依赖模型预训练阶段学到的泛化能力来解答。
   - **结合 CoT**：即 **Zero-shot CoT**。最经典的用法是在 Prompt 末尾加上一句魔法指令：“*Let's think step by step*（让我们一步一步地思考）”，以此激发模型自身的推理链。
2. **Few-shot（少样本提示）**：
   - **定义**：在 Prompt 中提供少量（通常 1~5 个）高质量的“输入-输出”示例（Demonstrations），让模型通过上下文学习（In-context Learning）掌握任务的模式和输出格式。
   - **结合 CoT**：即 **Few-shot CoT**。在提供的示例中，不仅包含问题和最终答案，还**必须包含完整的中间推理过程**。这样模型在回答新问题时，就会模仿示例中的逻辑步骤进行推导，这是目前激发大模型复杂推理能力最稳定有效的方法。
