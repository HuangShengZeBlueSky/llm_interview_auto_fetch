## 专题总结

## 本专题考什么
- **增量预训练（CPT）的核心动机**：领域知识注入、多语言能力扩展（如扩充中文词表）以及缓解灾难性遗忘。
- **预训练数据工程**：高质量语料的准备流程，特别是提升GPU计算效率的**数据拼接（Packing）**机制。
- **高效训练方案**：在算力受限情况下，使用 PEFT（如 LoRA）进行二次预训练的原理与工程实践。
- **大模型底层机制**：LLM 推理过程的阶段划分（Prefill 与 Decoding），考察对自回归生成机制的深度理解。

## 答题主线
- **需求出发**：通用大模型（如 LLaMA2）缺乏特定垂直领域知识或某类语言（如中文）能力，引出增量预训练的必要性。
- **数据与工程**：强调数据清洗与配比（通用数据+领域数据防遗忘），并详细阐述如何通过 Token 拼接（Packing）消除 Padding 带来的算力浪费。
- **训练策略**：根据算力预算，选择全量参数更新或基于 LoRA 的低成本二次预训练，明确各自的优化目标。
- **闭环到推理**：理解模型训练后的使用过程，清晰拆解输入处理（Prefill）与自回归生成（Decoding）的底层差异。

## 代表题答法

### 问题1：为什么要增量预训练？需要做哪些准备工作？
**答法要点：**
- **为什么做**：
  1. **注入领域知识**：通用模型缺乏医疗、金融等垂直领域的深度知识。
  2. **扩展语言能力**：例如 LLaMA2 中文语料占比极低，需要通过增量预训练提升中文理解与生成能力。
  3. **知识更新**：更新模型内部过时的世界知识。
- **准备工作**：
  1. **词表扩充（可选）**：如果目标语言 Token 压缩率低（如用 LLaMA 原生词表切分中文），需训练并合并新词表。
  2. **数据清洗与配比**：收集高质量领域语料，进行去重、去毒；**关键点**：必须混入一定比例（如 10%-20%）的通用语料（如 Wikipedia），以缓解**灾难性遗忘（Catastrophic Forgetting）**。
  3. **数据格式化**：Tokenize 并进行数据拼接（Packing）。

### 问题2：Pretrain阶段，为什么需要数据拼接（Packing）？有哪些拼接方式？
**答法要点：**
- **为什么需要拼接**：大模型训练通常采用固定的上下文长度（如 4096）。真实文档长度不一，如果短文档用 Padding 填充到 4096，会产生大量无效计算，严重浪费 GPU 算力。拼接可以保证每个 Batch 的有效 Token 占比接近 100%。
- **拼接方式**：
  1. **直接拼接（传统方式）**：将多篇文档用 `&lt;EOS&gt;` 符连接，直到填满 Context Length。缺点是计算 Attention 时不同文档之间会产生交叉注意力（Cross-contamination），引入噪声。
  2. **带掩码的拼接（Document Masking / Block Diagonal Attention）**：在拼接的基础上，修改 Attention Mask 矩阵（呈块状对角阵），使得同一 Sequence 内的不同文档在计算 Attention 时互相不可见，既提升了硬件利用率，又保证了训练数据的纯净度。

### 问题3：为什么需要对 LLaMA2 做基于 LoRA 的二次预训练？其目标是什么？
**答法要点：**
- **为什么用 LoRA 做预训练**：
  1. **算力限制**：全参数增量预训练成本极高，普通团队难以承担。
  2. **参数高效**：LoRA 通过在 Linear 层旁路注入低秩矩阵，仅需更新极少量的参数（通常 < 1%），大幅降低显存占用（可结合 ZeRO 优化），使得在消费级/少量专业级显卡上完成预训练成为可能。
- **核心目标**：
  1. **低成本语言/领域适配**：让模型快速学习新语言（如中文）的语法结构、词汇分布，或掌握垂直领域的专业术语。
  2. **保持通用能力**：冻结了原模型的主干参数，天然具备较强的抗灾难性遗忘能力，能在保留 LLaMA2 原有强大逻辑推理能力的基础上，为其“外挂”新的知识库。

### 问题4：大模型的推理过程分哪些阶段？Decoding 阶段的特点是什么？
**答法要点：**
- **推理过程分为两个核心阶段**：
  1. **Prefill（预填充阶段）**：处理用户的输入 Prompt。模型并行计算输入序列中所有 Token 的 Attention，生成并缓存第一组 KV Cache，输出第一个预测的 Token。此阶段是**计算密集型（Compute-bound）**。
  2. **Decoding（解码阶段/递归推理）**：自回归生成阶段。
- **Decoding 阶段的特点**：
  1. **逐字生成**：将上一步生成的 Token 作为当前步的输入，结合历史缓存的 KV Cache，预测下一个 Token，直到遇到 `&lt;EOS&gt;`。
  2. **访存密集型（Memory-bandwidth bound）**：每次只处理 1 个 Token，计算量小，但需要频繁从显存中读取庞大的 KV Cache 和模型权重，推理速度受限于显存带宽。
  3. **优化方向**：针对 Decoding 的瓶颈，工业界常采用 PagedAttention（vLLM）、FlashDecoding 等技术来优化显存访问与分配。
