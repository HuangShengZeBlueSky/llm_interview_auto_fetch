## 代表题答法

### 问题1：SFT（有监督微调）的数据集格式？
**答题要点：**
SFT 数据集通常以 `JSONL` 格式存储，核心是“指令-响应”对。主流格式分为两种：
1. **经典 Alpaca 格式（单轮为主）**：
   包含 `instruction`（任务指令）、`input`（可选的上下文或输入数据）、`output`（期望的模型回答）。
   ```json
   {"instruction": "翻译以下句子", "input": "Hello world", "output": "你好，世界"}
   ```
2. **OpenAI / ChatML 格式（适配多轮对话）**：
   采用 `messages` 列表，通过 `role`（system, user, assistant）区分角色，目前工业界更推荐此格式，便于拼接多轮上下文。
   ```json
   {"messages": [{"role": "system", "content": "你是一个助手"}, {"role": "user", "content": "你好"}, {"role": "assistant", "content": "你好！请问有什么可以帮您？"}]}
   ```

### 问题2：RM（奖励模型）的数据格式？
**答题要点：**
RM 的目标是学习人类偏好，因此数据格式是**偏序对（Pairwise）**，即针对同一个 Prompt 的两个不同回答进行打分对比。
核心字段通常包含：
- `prompt`：用户的输入指令（可能包含历史对话）。
- `chosen`：人类或强模型偏好的、质量更高的回答（正样本）。
- `rejected`：质量较差、被拒绝的回答（负样本）。
```json
{
  "prompt": "如何做西红柿炒鸡蛋？",
  "chosen": "第一步准备食材...第二步热锅下油...",
  "rejected": "西红柿炒鸡蛋很好吃。"
}
```
*加分项：可以补充说明，在实际训练中，RM 模型会将 chosen 和 rejected 分别输入模型计算标量分数，并通过 Margin Loss 优化，使得 chosen 的得分尽可能高于 rejected。*

### 问题3：SFT数据集如何生成？
**答题要点：**
工业界构建 SFT 数据集通常采用“三管齐下”的策略：
1. **开源数据集清洗**：
   收集高质量开源数据（如 ShareGPT、FLAN、UltraChat 等），进行格式统一、去重、去毒和语言过滤。
2. **强模型合成（知识蒸馏）**：
   - **Self-Instruct 范式**：利用少量人工编写的种子指令（Seed Prompts），调用 GPT-4 等强模型自动生成更多指令和对应的回复。
   - **Evol-Instruct（指令进化）**：在基础指令上，通过提示词让 GPT-4 增加约束、深化逻辑，生成更复杂、难度更高的数据（如 WizardLM 的做法）。
3. **人工专家标注（核心壁垒）**：
   针对特定垂直领域（如医疗、法律、代码）或对齐人类价值观（Safety）的数据，依赖领域专家进行精细化撰写和校对，这部分数据量虽小，但对模型最终的语气和专业性起决定作用。
*总结：生成后必须经过严格的后处理，包括基于规则的过滤（长度、特定词汇）和基于模型的过滤（用 RM 或 GPT-4 打分剔除低质量数据）。*
