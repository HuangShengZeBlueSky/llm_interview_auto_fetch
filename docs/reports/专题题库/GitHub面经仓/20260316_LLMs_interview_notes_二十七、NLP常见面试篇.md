## 专题总结

## 本专题考什么
本专题涵盖了NLP（自然语言处理）领域的经典任务、核心算法以及工程落地中的常见痛点。作为综合面试篇，主要考察候选人以下三个维度的能力：
- **基础任务与Pipeline**：文本分类、序列标注（NER）、文本摘要等经典任务的定义、标准流程及应用场景。
- **经典模型与机制**：对特定算法底层逻辑的理解，如CRF的全局序列约束、Pointer-generator的Copy机制、Annoy的近似最近邻检索。
- **工程痛点与解决方案**：在实际业务中遇到“长文本限制”、“类别不平衡”、“OOV（未登录词）”等问题时的系统性解决思路。

## 答题主线
面对这类综合性NLP问题，建议采用**“痛点驱动 -> 分层拆解”**的逻辑进行回答：
- **遇到工程痛点（如类别不平衡、长度限制）**：按“数据层 -> 模型层 -> 损失函数层/评估层”的结构化思路给出多套方案。
- **遇到具体算法（如CRF、Pointer-generator、Annoy）**：按“它是什么 -> 解决了什么痛点 -> 核心机制是什么”的逻辑展开，突出该算法的不可替代性。
- **遇到任务流程（如文本分类）**：按工业界标准的Pipeline（数据清洗-特征提取-模型预测-后处理）进行闭环描述。

## 代表题答法

### 问题1：怎么处理类别不平衡？
**答题思路**：从数据、模型、损失函数和评估指标四个维度全面作答。
- **数据层面**：对少数类进行过采样（或数据增强，如EDA、回译、大模型生成），对多数类进行欠采样。
- **模型层面**：采用集成学习（如将多数类切分后与少数类组合训练多个模型后Voting）。
- **损失函数层面**：引入代价敏感学习（Cost-sensitive learning），如调整类别权重（Class Weights），或使用 **Focal Loss**（挖掘难分样本）、**Dice Loss**（针对极度不平衡的NLP任务）。
- **评估指标**：明确指出在不平衡场景下不能看Accuracy，必须看宏观/微观 F1-score、Precision、Recall 或 PR-AUC 曲线。

### 问题2：如何解决模型输入长度限制问题？
**答题思路**：区分传统工程截断方案与大模型时代的底层优化方案。
- **工程截断/拼接法**：将长文本切片（Chunking），分别输入模型提取特征，最后通过 Pooling（如Mean/Max）或 Attention 融合；或者先用摘要模型压缩文本，再输入主模型。
- **注意力机制优化**：将全局Attention（$O(N^2)$复杂度）替换为稀疏注意力（如 Longformer 的滑动窗口+全局节点，BigBird 的随机+局部+全局注意力）或线性注意力。
- **位置编码外推（大模型常用）**：采用相对位置编码（如 ALiBi）或对 RoPE（旋转位置编码）进行位置插值（PI）、NTK-aware 缩放，以提升模型在推理时的长文本外推能力。
- **系统级优化**：提及 Ring Attention 或 FlashAttention，通过分块计算和显存优化支持更长上下文。

### 问题3：文本分类的具体流程与应用场景？
**答题思路**：体现工业级Pipeline的完整性。
- **应用场景**：情感分析（舆情监控）、意图识别（对话系统）、垃圾邮件/评论拦截、新闻主题分类等。
- **具体流程**：
  1. **数据预处理**：清洗（去噪、去停用词）、分词（Tokenization，如BPE/WordPiece）。
  2. **特征表示**：从传统的 TF-IDF、Word2Vec 到目前的预训练语言模型（BERT/RoBERTa）提取句向量（如 [CLS] token）。
  3. **模型构建**：根据算力与需求选择 TextCNN、TextRNN、Transformer 或直接微调大模型。
  4. **输出与损失**：全连接层 + Softmax（多分类）或 Sigmoid（多标签），计算 CrossEntropy Loss。
  5. **评估与部署**：计算 F1 等指标，进行模型压缩（蒸馏/量化）后上线。

### 问题4：抽取式摘要和生成式摘要存在哪些问题？Pointer-generator 解决了什么？
**答题思路**：对比两种摘要的优缺点，引出 Pointer-generator 的核心贡献。
- **抽取式问题**：直接从原文摘取句子，连贯性差，缺乏总结性，容易显得生硬。
- **生成式问题**：容易产生 OOV（未登录词）问题，且存在严重的“幻觉（Hallucination）”和“重复生成（Repetition）”现象。
- **Pointer-generator 的作用**：
  - **解决OOV/提升准确性**：引入了 Copy 机制（指针网络）。通过计算一个生成概率 $p_{gen}$，在每一步动态决定是“从词表中生成一个词”还是“从原文中拷贝一个词”。
  - **解决重复生成**：引入了 Coverage 机制（覆盖率惩罚），记录之前已经关注过的原文位置，惩罚模型再次对同一位置分配高注意力，从而减少重复。

### 问题5：什么是CRF？它的核心思想与三个基本问题是什么？
**答题思路**：精准定位CRF在序列标注中的“规则约束”作用。
- **核心思想**：CRF（条件随机场）是一种判别式无向图模型。在NER等序列标注任务中，单纯的深度学习模型（如BiLSTM）只考虑输入特征，而CRF通过**转移矩阵**学习相邻标签之间的约束关系（例如 `I-ORG` 前面必须是 `B-ORG` 或 `I-ORG`），从而保证全局输出序列的合法性。
- **三个基本问题**：
  1. **评估（概率计算）问题**：给定模型参数和输入序列，计算某个特定输出序列的条件概率（通常用前向-后向算法）。
  2. **解码（预测）问题**：给定模型参数和输入序列，求概率最大的输出序列（使用 **Viterbi 维特比算法**，这是面试常考点）。
  3. **学习（训练）问题**：给定训练数据，通过极大似然估计（MLE）或梯度下降法求解模型参数（转移概率矩阵和发射概率矩阵）。

### 问题6：Annoy 算法介绍及应用？
**答题思路**：结合当前热门的 RAG（检索增强生成）或向量检索背景作答。
- **它是什么**：Annoy (Approximate Nearest Neighbors Oh Yeah) 是 Spotify 开源的一种基于树结构的**近似最近邻（ANN）检索算法**。
- **核心机制**：通过随机选择两个点并作其等距超平面，将高维向量空间不断二分，构建出一棵二叉树。为了提高召回率，会构建多棵这样的树形成“森林”。检索时，在树中快速定位到目标向量所在的叶子节点，并在邻近节点中寻找最相似的向量。
- **应用场景**：在 NLP 中常用于海量文本向量的快速召回（如语义搜索、推荐系统、RAG中的知识库检索），它通过牺牲极少量的精度，换取了检索速度和内存占用的极大提升。
