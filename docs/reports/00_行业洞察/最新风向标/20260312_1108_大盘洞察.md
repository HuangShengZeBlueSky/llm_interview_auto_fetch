## 🔥 二、 高频考点红榜

结合本周面经，以下四大硬核考点霸榜，其背后折射的是大厂当前最迫切的业务焦虑：

### 1. 🥇 显存刺客终结者：KV Cache & GQA & FlashAttention
*   **业务背景**：大模型商业化落地的最大阻碍不是算力（FLOPs），而是**显存带宽（Memory Wall）**。在自回归生成中，长文本带来的 KV Cache 呈线性爆炸，直接卡死并发量（Batch Size）。
*   **为什么火**：GQA（分组查询注意力）通过共享 KV 头在物理层面压缩显存；FlashAttention 通过硬件感知（SRAM 与 HBM 的读写级差），利用 Tiling（分块）和 Online Softmax 彻底消灭了 $O(N^2)$ 的中间注意力矩阵。**不懂这套组合拳，在大厂根本无法做模型部署与推理优化。**

### 2. 🥈 长文本续命神药：RoPE（旋转位置编码）与外推插值
*   **业务背景**：Kimi、Claude 掀起的“超长上下文之战”是目前的行业主旋律。模型从 8K 扩展到 128K 甚至 1M，重新训练成本极高。
*   **为什么火**：RoPE 巧妙地用绝对位置的复数旋转实现了相对位置的表达。大厂极度关注候选人是否懂得如何通过 **线性插值（Linear PI）、NTK-aware（高频保留，低频插值）或 YaRN** 等技术，在不破坏模型原有能力的前提下，平滑解锁超长文本能力。

### 3. 🥉 对齐与进化的艺术：DPO & 合成数据 (Synthetic Data)
*   **业务背景**：高质量人类标注数据（SFT）已近枯竭，且 SFT（行为克隆）无法教会模型“什么是更好的回答”。传统 RLHF（PPO）又因为需要维护四个模型，训练极度不稳定。
*   **为什么火**：DPO（直接偏好优化）用极其优雅的数学转换，直接在语言模型上优化交叉熵，干掉了 Reward Model。同时，利用强模型（如 GPT-4）进行 **Evol-Instruct（指令进化）** 构造复杂合成数据，已成为各家大厂弯道超车的核心护城河。

### 4. 🏅 垂直落地的最后一公里：GraphRAG & 灾难性遗忘
*   **业务背景**：B端客户（医疗、法律）对幻觉零容忍。传统向量 RAG 只能解决“点状”事实检索，面对“总结这批卷宗的核心脉络”这种宏观多跳问题直接抓瞎。同时，微调垂直大模型极易把通用能力“洗掉”。
*   **为什么火**：GraphRAG 通过引入知识图谱和社区摘要（Community Summaries），实现了全局视角的 Map-Reduce 问答；而在微调时，通过 **Data Replay（数据混合）、LoRA 路由或 EWC 正则化** 抵抗灾难性遗忘，是业务落地的必考题。

## ⚔️ 三、 能力代差分析

在当前的面试修罗场中，面试官最喜欢用以下**“连环追问”**来精准区分“普通候选人（背题家）”和“高级候选人（实战派）”：

*   **代差 1：从“懂公式”到“懂硬件” (Hardware-Awareness)**
    *   *普通候选人*：能背出 FlashAttention 降低了显存，时间复杂度还是 $O(N^2)$。
    *   *高级候选人（追问）*：**“既然复杂度没变，为什么 FA 在长文本下还能加速？”** 高级候选人能一针见血地指出 GPU 的 Memory-bound 本质，讲透 Online Softmax 中局部最大值 $m$ 和指数和 $l$ 的增量更新机制，以及反向传播时的 Recomputation（重计算）如何用算力换显存带宽。
*   **代差 2：从“调包微调”到“深刻理解参数空间”**
    *   *普通候选人*：知道 LoRA 是冻结主干，旁路加 A、B 两个低秩矩阵。
    *   *高级候选人（追问）*：**“A 和 B 的初始化方式能互换吗？如果 LoRA 导致了灾难性遗忘，底层原因是什么？”** 高级候选人能指出 B 必须全零初始化以保证微调初期的等价性；并深刻洞察到如果 Rank 设置过大或 LR 过高，LoRA 激活值会剧烈改变 Transformer 层的输出分布，依然会导致旧知识崩塌。
*   **代差 3：从“跑通 Demo”到“工业级鲁棒性”**
    *   *普通候选人*：知道 RAG 就是切块（Chunking）+ 向量检索 + LLM 生成。
    *   *高级候选人（追问）*：**“法律条文场景下，按字数切块会导致什么灾难？Function Call 时大模型瞎编 API 参数怎么兜底？”** 高级候选人会抛出 Rule-based Chunking（保留法条层级元数据）、混合检索，以及在 Agent 工作流中引入 Schema Validation 和 Error Message 传回的 Self-Correction（自我修正）机制。

## 🚀 四、 行动建议

对于近期准备冲击大厂大模型算法岗的系统用户，我给出以下**复习优先级与战术建议**：

    *   **避坑**：不要再花大量时间看 BERT 时代的 Encoder 架构，全面拥抱 Decoder-only 及其 Scaling Law 信仰。
2.  **🟡 P1 级（分水岭）：掌握 Post-Training 的数据与算法闭环**
    *   **必做**：深入理解 SFT 与 DPO 的本质区别；准备一个关于“如何构造/清洗高质量微调数据（如难负样本挖掘、Evol-Instruct）”的实战 Story；掌握缓解灾难性遗忘的 3 种以上工业界做法。
3.  **🟢 P2 级（加分项）：前沿应用与跨域结合**
    *   **必做**：了解 GraphRAG 的 Indexing 和 Querying 流程；熟悉 Agent 的 Function Call 训练与推理闭环。如果背景是推荐系统，务必准备“LLM 如何赋能传统搜广推（如特征生成、交互式推荐）”的交叉命题。
