# 题目1：实习微调过Qwen2，你说说Qwen2的模型结构吧，Qwen2相比Qwen1做了哪些改进

**1. 题目描述：**
简述 Qwen2 的底层模型架构，并详细对比其相较于第一代 Qwen 模型的核心升级与改进点。

**2. 前置基础知识：**
必须了解 Transformer 的 Decoder-only 架构、主流的注意力机制变体（MHA, MQA, GQA）、位置编码（RoPE）、激活函数（SwiGLU）以及大模型多语言支持与词表大小（Vocabulary Size）的关系。

**3. 核心解答：**
*   **Qwen2 模型基本结构：** 
    基于主流的 Decoder-only 架构。核心组件包括：旋转位置编码 (RoPE)、SwiGLU 激活函数、RMSNorm 归一化（且采用 Pre-Norm 结构）、以及 Grouped Query Attention (GQA)。
*   **相比 Qwen1 的核心改进：**
    1.  **全面普及 GQA (Grouped Query Attention)：** Qwen1 只在部分大参数模型使用了 GQA，而 Qwen2 在全系列（包括 0.5B、1.5B 等小模型）中均采用了 GQA。这大幅度降低了推理时的 KV Cache 显存占用，显著提升了推理速度和吞吐量。
    2.  **上下文长度的大幅扩展：** Qwen1 的上下文长度通常在 8K 到 32K 之间；Qwen2 通过改进的 RoPE 扩展技术（如 YARN 等机制），将全系列模型的上下文长度大幅扩展至 128K 甚至更高。
    3.  **多语言与代码/数学能力增强：** 优化了训练数据配比，极大地增强了除中英以外的 27 种语言能力，并在代码和数学推理任务上的表现实现了跨越式提升。
    4.  **Tie Word Embeddings (权重共享)：** Qwen2 针对部分模型采用了 Tie Embeddings 技术（即输入词嵌入层与输出层 lm_head 共享权重），在保持模型性能的同时有效减少了参数量。

**4. 引申思考/面试追问：**
*   *追问：* 解释一下 GQA 相比于标准的 MHA (Multi-Head Attention) 和 MQA (Multi-Query Attention) 的区别和优势？
*   *思路：* MHA 每个 Query 头都有独立的 K 和 V 头（显存消耗大）；MQA 所有 Query 头共享一组 K 和 V 头（显存最小，但性能有损）；GQA 是折中方案，将 Query 分组，每组共享一组 K 和 V，在保证接近 MHA 性能的同时，实现了接近 MQA 的推理加速和低显存占用。

# 题目2：为什么现在 Decoder-only 成为大模型的主流架构，相比于 Encoder-Decoder 的架构好在哪里

**1. 题目描述：**
分析当前大语言模型普遍放弃 Encoder-Decoder 架构，转而采用单向 Decoder-only 架构的根本原因及其工程与算法优势。

**2. 前置基础知识：**
掌握自回归生成（Autoregressive Generation）的原理，理解双向注意力（Bidirectional Attention，如 BERT）与因果注意力（Causal Attention，如 GPT）的区别，了解 In-context Learning 的机制。

**3. 核心解答：**
现在 Decoder-only 成为主流，主要基于以下三个维度的优势：
*   **算法与泛化能力维度 (Zero-shot / In-context Learning)：** Decoder-only 采用严格的 Next Token Prediction（下一个词预测）作为训练目标。研究表明，这种自回归的单向语言建模范式，迫使模型将所有的上下文理解压缩到当前的隐状态中，赋予了模型极强的 In-context Learning（上下文学习）和 Zero-shot 泛化能力。相比之下，Encoder-Decoder 架构通常在特定的微调任务（如翻译、摘要）上表现好，但通用泛化能力较弱。
*   **工程与计算效率维度 (KV Cache 与 Attention 算子)：** 
    *   在推理阶段，Encoder-Decoder 架构需要维护两套机制：Encoder 的双向 Attention 和 Decoder 的 Cross-Attention + Causal Attention，导致 KV Cache 管理极其复杂。
    *   Decoder-only 架构结构高度统一，只有一种 Causal Self-Attention，极大地简化了底层算子优化（如 FlashAttention 极其契合 Decoder-only）和显存管理（如 PagedAttention）。
*   **Scaling Law 的验证：** 早期学术界认为"双向注意力才能更好理解上下文"。但 OpenAI 通过 GPT 系列证明了，只要参数量和数据量足够大（Scale up），单向注意力模型完全可以隐式地学到极好的上下文表示。当模型足够大时，架构的微小差异被庞大的数据和算力抹平，而 Decoder-only 凭借其极高的训练效率胜出。

**4. 引申思考/面试追问：**
*   *追问：* 既然 Decoder-only 这么好，像 GLM 这种 PrefixLM 架构（前半段双向，后半段单向）相比纯 CausalLM 有什么优缺点？
*   *思路：* PrefixLM 的 Prompt 部分使用双向注意力，理论上对 Prompt 的理解更深，适合复杂的 NLU 任务；但缺点是破坏了架构的绝对统一性，在接入 FlashAttention 等针对 CausalLM 极度优化的底层算子时，需要做额外的适配和魔改。

# 题目3：讲一下 RoPE，目前常用的位置编码还有哪些，RoPE有什么优点

**1. 题目描述：**
解释旋转位置编码（RoPE）的数学思想与优势，并列举业界其他主流的位置编码方案。

**2. 前置基础知识：**
了解绝对位置编码（Absolute PE，直接加在输入词向量上）和相对位置编码（Relative PE，加在 Attention 权重计算上）的区别。熟悉复数乘法在几何上的意义（复平面上的旋转）。

**3. 核心解答：**
*   **目前常用的位置编码：**
    1.  *绝对位置编码：* Sinusoidal（正余弦位置编码，Transformer原论文使用）、Learned PE（可学习位置编码，BERT使用）。
    2.  *相对位置编码：* ALiBi（带有线性偏置的注意力，通过距离衰减 Attention Score）、T5 Relative Bias。
*   **RoPE (Rotary Position Embedding) 原理：**
    RoPE 的核心思想是 **"通过绝对位置编码的计算方式，实现相对位置编码的效果"**。它借助欧拉公式，将 Token 的位置信息映射为复平面上的一个旋转角度。在进行 Attention 计算前，先对 Query 和 Key 向量根据其绝对位置 $m$ 和 $n$ 进行相应的旋转。当计算 Q 和 K 的内积时，由于复数内积的性质，结果只与它们的相对角度差异（即相对位置 $m-n$）有关。
*   **RoPE 的优点：**
    1.  **兼顾绝对与相对优势：** 既保留了绝对位置信息（方便模型感知全局序列），又在 Attention 打分时天然体现了相对位置距离，符合语言学的局部依赖特性。
    2.  **无需修改 Attention 矩阵：** 相比于传统的相对位置编码需要修改 Attention 矩阵的计算图，RoPE 只需要在 Q 和 K 相乘之前做一次轻量级的旋转变换（Hadamard 积），计算效率极高。
    3.  **良好的长度外推性 (Extrapolation)：** 配合动态 NTK-aware 缩放、YaRN 等插值技术，RoPE 能够非常平滑地将模型扩展到训练时未见过的超长上下文窗口。

**4. 引申思考/面试追问：**
*   *追问：* 当需要把基于 RoPE 的模型上下文从 8K 扩展到 32K 时，直接推理会崩掉，为什么？常用的插值方法有哪些？
*   *思路：* 直接推理崩掉是因为模型遇到了训练时从未见过的旋转角度（高频特征越界）。常用方法有：线性插值（Linear PI，把 32K 的位置映射回 8K 内）、NTK-aware 插值（高频不插值，低频插值，保留局部细节的同时扩展全局）。

# 题目4：为什么有了 SFT 之后还需要 RLHF

**1. 题目描述：**
阐述在完成有监督微调（SFT）后，大模型仍需引入基于人类反馈的强化学习（RLHF）的根本原因。

**2. 前置基础知识：**
理解 SFT 的损失函数（交叉熵/Teacher Forcing）及其局限性。理解大模型对齐（Alignment）的 3H 目标（Helpful, Honest, Harmless）。掌握 Exposure Bias（暴露偏差）概念。

**3. 核心解答：**
有了 SFT 后仍需要 RLHF，主要解决以下三个核心痛点：
*   **训练目标不一致 (Objective Mismatch)：** SFT 的本质是“模仿”，其损失函数（Next Token Prediction）追求的是在训练数据分布上最大化似然概率。但人类真实的需求并不是“预测下一个词”，而是要求回答“有用、诚实、无害”。RLHF 通过引入奖励模型（Reward Model），直接将人类的价值观和主观偏好转化为标量奖励，去直接优化模型策略，实现了目标对齐。
*   **缓解暴露偏差 (Exposure Bias) 与幻觉：** SFT 采用 Teacher Forcing 模式训练，模型每一步预测都基于完美的 Ground Truth。但在实际推理时，模型是自回归生成的，一旦前面生成了错误的 Token，误差就会累积且模型不知道如何纠正（产生幻觉）。RLHF 允许模型在自己生成的轨迹（Trajectory）上进行探索，通过强化学习的奖惩机制，模型能学会如何避免错误路径，或在说错话后如何挽回。
*   **数据获取成本与利用率：** 构建高质量的 SFT 数据（完整且完美的回答）成本极高。而在 RLHF 中，训练奖励模型只需要人类对多个回答进行**排序（Preference Data）**。人类做判别题（哪个回答更好）比做生成题（写一篇完美文章）容易得多，成本更低。同时，偏好数据包含了丰富的“负面信号”，能明确告诉模型“不要做什么”。

**4. 引申思考/面试追问：**
*   *追问：* 在 RLHF 训练中，为什么要在 PPO 的 Reward 目标中加入一个与 SFT 模型的 KL 散度惩罚项？
*   *思路：* 为了防止“奖励黑客行为”（Reward Hacking）。如果不加限制，模型会找到奖励模型的漏洞，生成一些毫无意义但能骗取高分的乱码。KL 惩罚项强迫 RL 训练后的模型输出分布不能偏离初始的 SFT 模型太远，保证语言的连贯性和基础能力。

# 题目5：PPO和 DPO 的主要思想是什么，DPO 相比 PPO 的 RLHF 做了哪些改进，除此之外还知道哪些 RL 算法

**1. 题目描述：**
对比 PPO 与 DPO 在大模型对齐中的核心思想，阐明 DPO 的架构改进优势，并列举其他前沿的对齐算法。

**2. 前置基础知识：**
强化学习基础（Actor-Critic 架构、Policy Gradient）。Bradley-Terry (BT) 偏好模型。理解 KL 散度在对齐中的作用。

**3. 核心解答：**
*   **PPO (Proximal Policy Optimization) 的主要思想：**
    PPO 是一种在线策略（On-policy）的强化学习算法。在 RLHF 中，它通常需要同时加载 4 个模型：Actor（被训练的策略模型）、Reference（计算 KL 散度的参考模型）、Reward（奖励模型）和 Critic（价值模型，用于估计 Advantage）。PPO 的核心思想是通过截断机制（Clipping）限制每次策略更新的幅度，防止模型更新步子太大导致崩溃。
*   **DPO (Direct Preference Optimization) 的主要思想与改进：**
    *   *主要思想：* DPO 基于 Bradley-Terry 模型，通过精妙的数学推导，证明了**奖励函数可以被策略模型的隐式表达所替代**。因此，它直接跳过了显式奖励模型的训练，将强化学习的对齐过程转化为一个标准的有监督对比损失（交叉熵）。
    *   *相比 PPO 的改进点：*
        1.  **极大地降低了显存消耗：** DPO 抛弃了 Reward 模型和 Critic 模型，只需要加载 Policy 模型和 Reference 模型，显存占用直接减半。
        2.  **训练极其稳定：** PPO 涉及复杂的 Advantage 计算、价值函数拟合和海量超参数调优，极易崩溃。DPO 就是一个简单的分类 Loss，训练稳定性极高，无需复杂的超参数搜索。
*   **其他 RL / 对齐算法：**
    1.  **KTO (Kahneman-Tversky Optimization)：** 不需要成对的偏好数据（Chosen/Rejected），只需要针对单个回答的“点赞”或“踩”标签即可训练，大幅降低数据收集难度。
    2.  **ORPO (Odds Ratio Preference Optimization)：** 将 SFT 和 DPO 结合到一步完成，甚至连 Reference 模型都不需要了，进一步节省显存。
    3.  **Rejection Sampling (拒绝采样) / Best-of-N：** 离线生成多个回答，用 RM 打分选出最好的，直接作为 SFT 数据进行微调。

**4. 引申思考/面试追问：**
*   *追问：* 既然 DPO 这么好，是不是 PPO 就可以被彻底淘汰了？
*   *思路：* 不是。DPO 是一种离线算法，其上限受限于偏好数据集的质量，容易对静态数据集过拟合。PPO 是在线算法，具有强大的探索（Exploration）能力。在需要强逻辑推理、代码生成等具有明确客观规则的复杂任务上，PPO 的天花板依然远高于 DPO。

# 题目6：代码：23.合并K个升序链表

**1. 题目描述：**
给定一个包含 $K$ 个升序链表的数组，要求将它们合并为一个升序链表并返回。

**2. 前置基础知识：**
链表的基本操作，优先队列（Min-Heap / 最小堆）的原理与 API 使用，或者分治法（Divide and Conquer / 归并排序思想）。

**3. 核心解答：**
这道题最经典且在工业界最实用的解法是使用**优先队列（最小堆）**。
*   *思路：* 我们维护一个大小为 $K$ 的最小堆。首先将所有 $K$ 个链表的头节点压入堆中。每次从堆顶弹出一个节点（这就是当前所有链表中最小的节点），将其拼接到结果链表的末尾。然后，如果被弹出节点还有下一个节点，就将它的下一个节点再压入堆中。重复此过程直到堆为空。
*   *复杂度：* 时间复杂度为 $O(N \log K)$，其中 $N$ 是所有节点的总数，每次堆操作耗时 $\log K$。空间复杂度为 $O(K)$，堆最多同时容纳 $K$ 个节点。

*   *Python 代码实现：*
```python
import heapq

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def mergeKLists(lists):
    # 定义虚拟头节点
    dummy = ListNode(0)
    curr = dummy
    
    heap = []
    # 初始化堆：将每个链表的头节点加入堆中。
    # 为了避免 ListNode 对象之间无法比较（当 val 相同时），我们在元组中加入索引 i
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))
            
    while heap:
        val, i, node = heapq.heappop(heap)
        curr.next = node  # 接入结果链表
        curr = curr.next
        
        # 如果当前节点还有后续节点，继续压入堆中
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
            
    return dummy.next
```

**4. 引申思考/面试追问：**
*   *追问：* 除了优先队列，还有什么时间复杂度也是 $O(N \log K)$ 的解法？
*   *思路：* 分治法（Divide and Conquer）。将 $K$ 个链表两两配对合并，第一轮合并后剩下 $K/2$ 个，第二轮剩下 $K/4$ 个，直到剩下 1 个。利用合并两个有序链表的逻辑，递归或迭代实现。

# 题目7：了解 Qwen 的结构吗，微调的数据配比是怎样的

**1. 题目描述：**
简述 Qwen 的模型架构特点，并深入探讨大模型 SFT 微调阶段的数据混合比例（Data Mixture）策略。

**2. 前置基础知识：**
（Qwen 结构在题目1已解答，此处着重数据配比）。了解 SFT 数据的分类（通用指令、垂直领域、安全防护等），理解“灾难性遗忘”（Catastrophic Forgetting）的概念，以及“Less is More”在数据质量上的体现。

**3. 核心解答：**
*   **Qwen 结构简述：** 采用 Decoder-only 架构，使用 RoPE、SwiGLU 和 Pre-RMSNorm。其最大特色是拥有极高压缩率的超大词表（约 150K），对多语言（尤其是中文）极其友好，新版全面引入了 GQA 和 Tie Embeddings 以提升效率。
*   **微调的数据配比（Data Mixture）策略：**
    微调数据不是一股脑扔进去，必须经过科学配比，通常遵循以下原则：
    1.  **通用数据与垂类数据混合（防遗忘）：** 即使是做特定领域的微调（如医疗大模型），也**绝对不能**只使用医疗数据。通常需要掺杂 20% - 50% 的高质量通用指令数据（如数学、逻辑、通用问答、代码等）。如果不加通用数据，模型会迅速发生“灾难性遗忘”，丧失基础的对话和指令遵循能力，甚至变成只会输出医疗术语的“复读机”。
    2.  **高质量与多样性主导 (Less is More)：** 数据质量远比数量重要。几万条粗糙的爬虫数据不如几千条人工精标的 Instruction 数据。数据应涵盖单轮问答、多轮对话、长文本摘要、角色扮演等多种 Prompt 形式。
    3.  **安全与拒答数据 (Safety & Refusal)：** 必须配置 3% - 5% 左右的安全性数据（如诱导违法、暴力的 Prompt），并配以标准的拒绝回答模板，保证模型不越安全红线。
    4.  **长短文本配比：** 数据集中必须包含一定比例的长文本输入输出。如果微调数据全是短句，模型在推理时会产生“长度退化”，无论问什么都只回答一两句话。

**4. 引申思考/面试追问：**
*   *追问：* 如果在微调过程中发现 Loss 降得很低，但模型在实际测试时不仅垂类知识没学好，连正常说话都不会了，可能是什么原因？
*   *思路：* 典型的数据配比失衡或过拟合。可能是：1. 垂类数据占比过高且形式单一；2. 学习率过大或 Epoch 设置过多导致严重过拟合；3. 数据集中存在大量格式错误或截断的脏数据。
