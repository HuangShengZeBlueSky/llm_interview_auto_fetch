## 大模型（LLMs）训练集面

### 1. SFT（有监督微调）的数据集格式？

- 主标签：模型微调
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 大模型（LLMs）训练集面 / 未知](https://articles.zsxq.com/id_axtljtl0bsvw.html)

### 基础知识补充

- 数据集通常采用JSON或JSONL格式存储。
- 核心字段包含指令（instruction）、输入（input）和输出（output）。
- 多轮对话常采用messages列表形式，区分role和content。

### 详细解答

SFT（有监督微调）的数据集格式设计旨在清晰地向模型展示“在特定指令下应该输出什么”。结论是：SFT数据通常被结构化为问答对或对话历史的形式，主流格式包括Alpaca格式和ShareGPT格式。原理上，Alpaca格式是最基础的单轮指令微调格式，通常包含三个字段：instruction（任务描述）、input（可选的上下文信息）和output（期望的模型回答）。在训练时，instruction和input会被拼接为Prompt。随着多轮对话能力的普及，ShareGPT格式成为主流，它使用一个包含多个字典的列表（如messages或conversations），每个字典标明了发言者的角色（role: user 或 assistant）和具体内容（content）。这种格式能自然地表达多轮交互的上下文，训练时模型仅对assistant角色的内容计算交叉熵损失。

### 案例模拟

面试官：在处理多轮对话的SFT数据时，如何计算Loss？ 候选人：在多轮对话数据中，我们会将整个对话历史拼接成一个长序列输入给模型。但在计算Loss时，我们只对模型（Assistant）生成的Token计算交叉熵损失，而将用户（User）输入的Token以及系统提示词（System Prompt）的标签设置为-100（在PyTorch中，CrossEntropyLoss默认忽略-100的标签）。这样可以确保模型专注于学习如何回答，而不是去预测用户接下来会问什么。

### 2. RM（奖励模型）的数据格式？

- 主标签：强化学习与RLHF
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 大模型（LLMs）训练集面 / 未知](https://articles.zsxq.com/id_axtljtl0bsvw.html)

### 基础知识补充

- 数据集核心是同一Prompt下的成对比较（Pairwise Comparison）。
- 包含提示词（prompt）、偏好回答（chosen）和拒绝回答（rejected）。
- 有时会包含边际值（margin）字段以表示偏好程度的强弱。

### 详细解答

RM（奖励模型）的数据集格式专为对比学习设计，用于教会模型区分回答的质量高低。结论是：RM数据必须包含成对的偏好信息，最标准的格式是包含Prompt、Chosen和Rejected三个核心字段的JSONL文件。在具体结构上，prompt字段记录了用户的输入指令及可能的对话历史；chosen字段存放了人类标注员认为质量更高、更符合意图的回答；rejected字段则存放了质量较差的回答。在更精细的业务场景中，数据可能还会包含一个margin或score_diff字段，用于量化Chosen比Rejected好多少（例如，是略好还是碾压性优势），这有助于在损失函数中引入动态裕度（Dynamic Margin），提升RM的判别精度。此外，如果是从排序数据（如A>B>C）转化而来，会在预处理阶段被拆解为多个(Chosen, Rejected)对。

### 案例模拟

面试官：如果一条RM数据中，Chosen和Rejected的长度差异很大（比如Chosen很长，Rejected很短），会有什么隐患？ 候选人：这会导致严重的“长度偏见（Length Penalty/Bias）”。大模型很容易学到一个捷径：只要回答长，分数就高，而不是真正去理解语义质量。为了解决这个问题，我们在构建RM数据集时，会尽量控制Chosen和Rejected的长度相近。如果无法避免，我们会在训练时引入长度惩罚项，或者在数据集中故意混入一些“短但高质量”的Chosen和“长但低质量”的Rejected样本，强迫模型关注内容本身。

### 3. PPO（强化学习）的数据格式？

- 主标签：后训练与对齐
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 大模型（LLMs）训练集面 / 未知](https://articles.zsxq.com/id_axtljtl0bsvw.html)

### 基础知识补充

- PPO训练阶段的数据集格式最为简单，仅需提供Prompt。
- 不需要提供标准答案（Target）或偏好对。
- 数据集通常只包含一个prompt或query字段。

### 详细解答

PPO（强化学习）阶段的数据集格式在整个RLHF流程中是最精简的。结论是：PPO数据集只需要提供无标签的Prompt，因为回答是由策略模型实时生成的，质量评估由奖励模型（RM）自动完成。在原理上，PPO的目的是让模型在各种可能的输入场景下探索并优化策略。因此，数据集通常是一个简单的JSONL文件，每行只包含一个prompt（或query）字段，记录了用户的指令或对话历史。在训练过程中，环境（即训练脚本）会从数据集中采样一个Prompt输入给当前的Actor模型，Actor生成对应的Response，然后将Prompt+Response送入RM模型获取奖励分数，最后利用这个分数更新Actor的参数。为了保证模型泛化性，PPO的Prompt数据集通常需要覆盖极广的领域和意图分布。

### 案例模拟

面试官：PPO阶段只需要Prompt，那这些Prompt可以和SFT阶段的Prompt完全一样吗？ 候选人：可以有交集，但最好不要完全一样。SFT阶段的Prompt通常是为了教会模型特定格式和基础能力，数量相对较少（几万条）。而PPO阶段的目的是为了对齐人类偏好和提升泛化性，因此需要海量且分布更广的Prompt（可能几十万条），包括很多SFT阶段没见过的长尾问题或对抗性问题（Red Teaming）。如果只用SFT的Prompt，PPO很容易过拟合，导致模型在遇到新问题时表现不佳。

## 大模型（LLMs）LLM生成SFT数据方法面

### 4. 四、大模型微调数据集格式篇

- 主标签：模型微调
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 大模型（LLMs）LLM生成SFT数据方法面 / 未知](https://articles.zsxq.com/id_1gzdghj84f9f.html)

### 基础知识补充

- 预训练数据为海量纯文本，SFT数据为指令问答对。
- RM/DPO数据为成对偏好对比，PPO数据为纯Prompt集合。
- 数据格式的演进反映了模型从学说话到懂规矩的对齐过程。

### 详细解答

大模型在不同训练阶段的数据集格式差异，本质上反映了优化目标的演进。结论是：从预训练到SFT，再到对齐阶段（RM/DPO/PPO），数据格式从无监督的纯文本逐渐过渡到高度结构化的偏好反馈。具体而言：1. 预训练（PT）阶段：数据是海量的纯文本（如网页、书籍），格式通常是简单的长字符串，目标是预测下一个Token。2. 监督微调（SFT）阶段：数据转变为(Prompt, Response)对，通常采用ShareGPT的多轮对话格式，目标是教会模型遵循指令。3. 奖励模型（RM）或DPO阶段：数据升级为(Prompt, Chosen, Rejected)三元组，引入了人类的相对偏好，目标是建立价值判断标准。4. 强化学习（PPO）阶段：数据退化为纯Prompt集合，模型在这些Prompt上自由生成并接受RM打分，目标是在广阔状态空间中寻找最优策略。掌握这些格式是进行大模型工程实践的基础。

### 案例模拟

面试官：如果我想用一份包含Prompt和高质量Response的数据，同时做SFT和DPO，应该怎么处理？ 候选人：这份数据直接做SFT没问题。但要做DPO，必须构造出Rejected样本。我们可以采取几种策略：一是使用当前SFT模型对这些Prompt生成回答，如果生成的回答质量不如原数据中的Response，就可以将原数据作为Chosen，生成的作为Rejected；二是引入一个较弱的开源模型生成回答作为Rejected；三是对原Response进行随机的词语替换或逻辑破坏来人工制造Rejected。构造完成后，将其整理为(Prompt, Chosen, Rejected)格式即可用于DPO训练。

### 5. 一、SFT数据集如何生成？

- 主标签：模型微调
- 来源条数：1
- 答案生成方式：模型自动补全
- 来源：[LLMs_interview_notes / 大模型（LLMs）LLM生成SFT数据方法面 / 未知](https://articles.zsxq.com/id_1gzdghj84f9f.html)

### 基础知识补充

- 人工标注构建高质量指令微调数据集，确保多样性。
- 利用Self-Instruct等技术通过大模型自动生成数据。
- 结合开源数据集清洗与业务场景数据进行混合配比。

### 详细解答

SFT（监督微调）数据集的生成主要分为人工构建和自动生成两种方式。结论是：高质量的SFT数据是激发大模型指令遵循能力的关键。原理上，人工标注通常由领域专家编写Prompt和对应的标准回答，确保数据的准确性和安全性，但成本高昂。为了降低成本，工程上常采用Self-Instruct技术，利用能力更强的闭源模型（如GPT-4）根据少量种子指令自动扩充生成大量问答对。此外，还会收集开源数据集（如Flan、ShareGPT）并进行严格的清洗、去重和质量过滤。在实际工程中，需要权衡数据的数量与质量，通常采用“少量高质量人工数据+大量清洗后的自动生成数据”的混合策略，并注意不同任务类型（如对话、代码、逻辑推理）的合理配比。

### 案例模拟

面试官追问：“如果生成的SFT数据存在幻觉或质量参差不齐，如何处理？” 回答：“我们会引入多重过滤机制。首先使用规则过滤掉过短、包含特殊字符或重复的数据；其次，利用奖励模型或强力LLM进行质量打分，剔除低分数据；最后，通过聚类算法保证数据分布的多样性，避免模型在特定句式上过拟合。对于核心业务场景，必须保留人工抽检环节。”
